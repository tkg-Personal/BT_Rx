# BT_Rx 設計日記

## 作業方法メモ（黒子セッション引き継ぎ用）

### プロジェクト概要
- Bluetooth A2DP レシーバー（3.5mmジャック出力）のPCB設計プロジェクト
- 設計ツール: **Autodesk Fusion Electronics** (EAGLE系)
- 仕様書: `BT_Rx_仕様書.md` (v4 最終版) — 回路構成・BOM・接続詳細の全てが書かれている
- 実装手順: `実装方法.md` — PCB設計〜はんだ付け〜テスト手順

### 黒子の役割
- ユーザーがFusion Electronicsで回路図・PCB設計を進める際のナビゲーター
- スクリーンショットを撮って (`mcp__screenshot-server__take_system_screenshot`) 画面を確認し、仕様書と照合する
- 次にやるべき操作を具体的に指示する
- 仕様書との相違点を見つけたら指摘する

### 作業の進め方
1. **画面確認**: ユーザーにスクリーンショットを見せてもらう or 自分で撮る
2. **仕様書照合**: `BT_Rx_仕様書.md` の該当セクションと比較
3. **指示出し**: Fusion Electronicsの操作を具体的に伝える
4. **記録**: 進捗や相違点をこの `日記.md` に記録

### Fusion Electronics での操作メモ
- 部品配置: 右パネル「コンポーネントを配置」からライブラリ検索
- 値設定: 部品をダブルクリック → プロパティ → 「値(V)」欄に入力
- SnapEDAインポート品 (EEUFC1V101等): 値フィールドがロックされている → 部品名=値なのでそのままでOK
- 標準ライブラリにない部品: SnapEDA (snapeda.com) から .lbr をダウンロード → external フォルダに配置
- 配線: 「デザイン」タブ → NET コマンド でピン間を結線
- ネット命名: **NAMEコマンド**（`接続` > `名前`）でワイヤに名前を付ける。同名ネットは自動的に電気接続される
- ラベル: **LABELコマンド**はネット名を表示するだけ。ラベルを置いただけでは接続されない！
- ピンスナップ: NETを描く時、ピン上で**緑の〇**が出ることを必ず確認。出ないとピンに接続されない
- SnapEDA部品のグリッド: SnapEDAインポート品はピンが標準グリッド(0.1inch)に乗っていない場合あり。**Altキーを押しながら**クリックで細かいグリッドにスナップ可能
- ERC: 「検証」タブ → ERC実行

### ライブラリ管理
- Fusionの標準ライブラリ: 抵抗(CHIP-0603, CHIP-0402)、コンデンサ(C_CHIP-0402, C_CHIP-0805)、LED、ピンヘッダ等
- カスタムライブラリ (external/): BM83SM1-00AA.lbr, PESD5V0S1BA.lbr (SnapEDA)
- C6/C7用: lib_EEUFC1V101 (SnapEDA、Panasonic FC 100uF 35V 電解)
- 電解コンデンサは標準ライブラリにφ4mmが無かったためSnapEDAから取得

## 2026-03-15 (後半): ネット名修正・ERC全承認・回路図完成

### 問題と原因
ERC警告10件「ネット XX 上にはピンが1つだけです」が残っていた。

**根本原因**: ラベル（LABELコマンド）を配置しただけで、ワイヤに対して**NAMEコマンドでネット名を付けていなかった**。Fusion Electronicsでは：
- **LABEL** = ネット名を画面に表示するだけ（電気的効果なし）
- **NAME** = ワイヤに実際のネット名を設定する（同名ネットは自動的に電気接続される）
- NETコマンドでワイヤを描く際、ピン上に**緑の〇**が出ないとピンに接続されない

### 修正内容
1. 各ワイヤに対してNAMEコマンドで正しいネット名を設定
   - BAT, VBUS, SYS_PWR, RST 等
   - 「既に存在します」→ OKで結合
2. ERC再実行 → エラー0件
3. 残り警告10件 → 全て承認
4. **回路図完成**

### 学んだこと
- ラベルを置くだけでは接続にならない。必ずNAMEコマンドでネットを命名すること
- SnapEDA部品はグリッドがずれてることがあり、Altキーで細かいグリッドにスナップする必要がある
- 「ピンが1つだけ」警告 = そのネットに1つしかコンポーネントピンが繋がっていない
