## LABEL

**機能**:

 バスおよびネットにテキスト ラベルをアタッチします。

**構文**:

 LABEL [XREF] [orientation] ..

**マウス キー**:

 *中央*クリックは、画層を選択します。
 *右*クリックは、ラベルを回転します。
 *[Shift]+右*クリックは、回転方向を反転します。

**関連項目**: [NAME](ECD-CLI-N.html#name)、[BUS](ECD-CLI-B.html#bus)、[FRAME](ECD-CLI-F.html#frame)

バスまたはネットの名前は、ラベル コマンドを使用して回路図上の任意の位置に配置できます。バスまたはネットをクリックすると、関連するラベルがマウス カーソルにアタッチされ、回転、別のレイヤへの変更、または別の位置への移動を行うことができます。2 回目のマウス クリックによって、ラベルの位置が定義されます。

ラベルの方向は、[ADD](ECD-CLI-A.html#add) コマンドでリストされる通常の定義を使用してテキストで定義できます(R0、R90、など)。

バスおよびネットには、任意の数のラベルを付けることができます。

ラベルを「CHANGE TEXT」で変更することはできません。

ラベルはプログラムによってテキストとして処理されますが、値は適切なバスまたはネットの名前に対応します。NAME コマンドを使用してバスまたはネットの名前を変更すると、関連付けられているすべてのラベルの名前が自動的に変更されます。

SHOW コマンドを使用してバス、ネット、またはラベルを選択すると、接続されているすべてのバス、ネット、およびラベルがハイライト表示されます。

### クロスリファレンス ラベル

オプションのキーワード XREF が指定されている場合、ラベルは「クロスリファレンス」ラベルになります。クロスリファレンス ラベルをマルチシート回路図で使用して、特定のネットが表示される次のシートを示すことができます(これは、バスではなく、ネットに対してのみ機能します)。XREF キーワードは、主にスクリプトで使用します。通常、設定は、[CHANGE XREF](ECD-CLI-C.html#x45) コマンドで以前に設定した設定から取得されるか、パラメータ ツールバーの[外部参照]ボタンをクリックして取得します。相互参照ラベルが表示される形式は、[オプション]/[設定]/[その他]ダイアログで定義されている「外部参照ラベル形式」の文字列、または [SET](ECD-CLI-S.html#set) コマンドでコントロールできます。次のプレースホルダが定義されており、任意の順序で使用できます。

    
        | 
            %F | 
             | 
            ラベルの周囲へのフラグ境界の描画を有効化 | 
         |

        | 
            %N | 
             | 
            ネットの名前 | 
         |

        | 
            %S | 
             | 
            次のシートの番号 | 
         |

        | 
            %C | 
             | 
            次のシートの列 | 
         |

        | 
            %R | 
             | 
            次のシートの行 | 
         |

    

既定の形式文字列は「%F%N/%S.%C%R」です。定義済みのプレースホルダとは別に、他の任意の ASCII 文字を使用することもできます。

//列および行の値は、そのネットが表示される[フレーム](ECD-CLI-F.html#x65)が次のシートに存在する場合にのみ機能します。%C または %R を 使用していて、そのシートにフレームがない場合、疑問符(「?」)が表示されます。

シート上のネットの列と行を決定する場合、最初に列を、次にその列内の行を考慮します。ここでは、XREF ラベルが通常のラベルよりも優先され、通常のラベルはネット ワイヤよりも優先されます。上位シート番号の場合は左最上部のフィールドのフレーム座標が取得され、下位シート番号の場合は右最下部のフィールドのフレーム座標が使用されます。

クロスリファレンス ラベルの方向は、「上位」シート番号と「下位」シート番号のどちらを指すかを定義します。R0 または R270 の方向を持つラベルは、図面の右または下の境界を指すため、上位シート番号を参照します。したがって、R90 または R180 の方向を持つラベルは、下位シート番号を参照します。ラベルの方向が R0 または R270 で、アタッチ先のネットが上位シートに存在しない場合は、代わりに次の下位シートへの参照が表示されます(R90 および R180 の場合も同様)。ネットが現在のシートにのみ表示される場合、クロスリファレンスは表示されず、ネット名のみが(形式文字列に %F プレースホルダが含まれている場合はフラグ境界で囲んで)表示されます。

ネット ワイヤの端点に配置されるクロスリファレンス ラベルは、ワイヤに接続され、ワイヤがラベルとともに移動し、ラベルがワイヤとともに移動するようになります。

クロスリファレンス ラベルの形式文字列は、回路図図面ファイルに保存されます。

[CHANGE](ECD-CLI-C.html#change) コマンドまたはラベルの_[プロパティ]_ダイアログを使用して、クロスリファレンス ラベルを通常のラベルに変更します。

### レイヤを選択する

他のコマンド(たとえば LINE など)とは異なり、LABEL コマンドは前回使用した画層を自身で追跡します。これには、他のコマンドがどの画層に描画するかには関係なく、ラベルが常に正しい画層に描画されるようにするという利点があります。これの欠点は、次のようにスクリプトでレイヤを設定する通常の方法が

LAYER _layer_;
LINE (1 2) (3 4);

機能しないということです。レイヤは、LABEL コマンドが既にアクティブになっているときに選択する必要があります。これは、次のように行います

LABEL _parameters_
LAYER _layer_
_more parameters_;

LABEL 行が「;」で終了して**おらず**、LAYER コマンドが新しい行で始まっていることに注意してください。
コマンド

LABEL
LAYER _layer_;

は、後続の LABEL コマンドで使用するレイヤを設定します。

## LAUNCH

**機能**:

 Web ベースの 3D パッケージ ジェネレータおよび検索を起動します。

**構文**:

 LAUNCH PACKAGE3D-GENERATOR
  LAUNCH PACKAGE3D-WEB-SEARCH  

**関連項目**: [PACKAGE](ECD-CLI-P.html#package)、[COPY](ECD-CLI-C.html#copy)

このコマンドは、Web ベースの 3D パッケージ ジェネレータおよび検索を起動するために使用されます。このコマンドは、デバイス エディタでの新しいデバイス バリアントの作成([PACKAGE](ECD-CLI-P.html#package) コマンドと同様)、およびライブラリ エディタでのライブラリへのパッケージのインポート([COPY](ECD-CLI-C.html#copy) コマンドと同様)に使用できます。

## LAYER

**機能**:

アクティブな図面レイヤーを選択し、新しいレイヤーを作成、既存のレイヤーの名前を変更、またはデザインからレイヤーを削除します。LAYER コマンドは、どのレイヤー上にオブジェクトを描画するかをコントロールし、必要に応じてカスタムまたは業界標準のレイヤーを定義することで、回路図または PCB の構成、可視性、および構造を管理するために不可欠です。

**構文**:

 LAYER layer_number
  LAYER layer_name
  LAYER layer_number layer_name
  LAYER layer_id
  LAYER layer_id layer_name
  LAYER [??] -layer_number  

**関連項目**: [DISPLAY](ECD-CLI-D.html#display)

### 図面レイヤを選択する

LAYER コマンドにパラメータが 1 つある場合は、現在のレイヤ、つまりワイヤや円などを描画するレイヤを変更するために使用します。メニューから[レイヤー]を選択すると、ポップアップ メニューが表示され、目的のレイヤーに変更できます。コマンド ラインから入力する場合、'layer_number' は任意の有効なレイヤー番号、'layer_id' は任意の有効なレイヤーの英数字識別子です(下記の「[定義済みレイヤー カテゴリとレイヤー - 最新および旧形式](#predefined-layer-categories-and-layers---current-and-legacy)」の項を参照)。'layer_name' はポップアップ メニューに表示されるレイヤー名です。すべてのモードで使えるレイヤーは一部のみです。

**信号レイヤー**は、 **[デザイン ルール] / [レイヤー スタック マネージャ]** で定義されたもののみ使用できます。

### 画層を定義する

2 つのパラメータを持つ LAYER コマンドは、新しい画層の定義または既存の画層の名前の変更に使用されます。たとえば、コマンド プロンプトに次のように入力すると、

LAYER 101 SAMPLE;

レイヤー番号 101、レイヤー名 SAMPLE の新しいレイヤーが定義されます。またば、「layer_number」の代わりに「layer_id」を使用することもできます。以下はその例です。

LAYER c2 +3.3V;

このコマンドは、ID「c2」の信号レイヤーの名前を「+3.3V」に変更します。

#### その他の LAYER コマンドの例

LAYER 1;
レイヤー番号 1 をアクティブな図面レイヤーとして選択します。

LAYER 105 MyCustomLayer;
番号 105 の新しいレイヤーを作成し、「MyCustomLayer」という名前を付けます。

LAYER c4 Power-Ground;
ID「c4」のレイヤーの名前を「Power-Ground」に変更します。

ボードで指定されていないレイヤーがフットプリントに含まれている場合、これらのレイヤーは、パッケージをボードに追加する(ADD または REPLACE)とすぐにボードに追加されます。

定義済みの画層には特殊な機能があります。レイヤ名は変更できますが、それらの機能(カテゴリ、番号、または ID に関連するもの)は同じままです。

独自のレイヤーを定義する場合は、**100** より大きく、**255** より小さい番号、または「u」接頭辞付きの英数字レイヤー id (例:「**u100**」)を使用します。それより小さい番号は、将来の Fusion バージョンで特殊な目的で割り当てられる可能性があります。

#### 上/下のペア

レイヤーの上/下のペアには、mirror コマンドを使用して相互にミラー化するという特性があります(下記の「[定義済みレイヤー カテゴリとレイヤー - 最新および旧形式](#predefined-layer-categories-and-layers---current-and-legacy)」を参照)。カスタム目的のために定義済みレイヤー TestTop/TestBottom を使用する他に、次のようなカスタム ペアを定義することもできます。

LAYER 53 SomethingTop;
LAYER 54 SomethingBottom;

レイヤー ミラー機能は、次のレイヤー ペアでサポートされています: 53/54、55/56、57/58、59/60。この機能は慎重に使用してください。将来の Fusion バージョンでは、これらのペアの処理方法が変更される場合があります。

### 削除, 画層を

レイヤーを削除するには、LAYER コマンドにマイナス記号を付け、その後にレイヤー番号を指定します。以下はその例です。

LAYER -103;
このコマンドは、レイヤー番号 103 を削除しますが、レイヤーが空の場合のみ有効です。レイヤーが空でない場合、プログラムは「layer is not empty: #」というエラー メッセージを表示します。ここで「#」はレイヤー番号を示します。

LAYER -105;
レイヤー番号 105 を削除します(空の場合のみ)。

レイヤーを削除する際にエラー メッセージを非表示にするには(たとえば、レイヤーが空でない、または存在しない場合でも許容されるスクリプトで使用する場合など)、「??」オプションを使用します。

LAYER ?? -110;
レイヤー 110 を削除しようとしますが、レイヤーが空でないか存在しない場合はエラーを抑制します。

**注**: 現時点では、「layer_id」を指定してレイヤーを削除することはできません。削除にはレイヤー番号のみがサポートされています。

### 定義済みのレイヤー カテゴリとレイヤー - 最新および旧形式

**レイヤー カテゴリ**は、銅箔、機械、シルクスクリーン、シミュレーション、接続性、ドキュメントなど、デザイン内での機能的役割に基づいてレイヤーをグループ化します。この分類により、複雑な回路図や PCB デザインに存在するさまざまなタイプのレイヤーを迅速に特定、フィルタ、および管理できるようになります。レイヤーをカテゴリに整理することで、ユーザは次のことができます。

- 編集または表示のために関連するレイヤーを簡単に見つけることができます。

- 可視性や編集アクションをグループ全体に適用できます。

- 各レイヤーの目的を一目で把握できます。

- 多数のレイヤーを含む大規模なデザイン作業時の混乱を軽減します。

レイヤー カテゴリにより、ナビゲーションとレイヤー管理が合理化され、デザインが複雑になるにつれて明確さとコントロール性能を維持しやすくなります。

#### レイアウトおよびパッケージ エディタ

**重要**: レイヤー名が業界用語により密接に対応するよう更新され、複雑なデザインをより効率的に整理およびナビゲートできるよう、レイヤー カテゴリが追加されました。**この変更は、旧形式のファイルには影響しません。**製品を使用して作業するときに、新しい名前、古い名前、またはそれらの組み合わせが表示される場合があります。

### レイヤー グループ / セット

Fusion には、プリセットレイヤー グループとユーザ定義レイヤー グループの 2 タイプのレイヤー グループがあります。これらは回路図モードとボード エディタ モードの両方で使用できますが、2 つのモード間で同時に表示することはできません。プリセット グループは eagle.scr であらかじめ定義されており、Fusion エディタではなく、その場所からのみ編集、名前変更、削除することができます。

ユーザ定義のレイヤー グループは以前から Fusion で利用可能で、現在ではレイヤー表示ダイアログのポップアップ メニューから直接アクセスして、より簡単に管理できるようになっています。

新しいレイヤー セットを作成するには、まずグループに含めるレイヤーを選択し、[新規追加]ボタンをクリックします。レイヤー セットの名前を入力して[OK]をクリックすると、レイヤー セットが作成されます。回路図エディタとボード エディタの両方で、最大 9 つのプリセットを作成できます。

レイヤー プリセットに名前を付ける際は注意してください。特定の名前は DISPLAY コマンドの動作に影響する場合があります。たとえば、レイヤー プリセットに「top」という名前を付けると、通常はトップ レイヤーのみを表示する既定の DISPLAY の動作が置き換えられ、プリセットに含まれるすべてのレイヤーが表示されます。

ユーザ定義レイヤー グループを削除するには、削除するグループをドロップダウン メニューから選択し、[削除]ボタンをクリックします。

## LINE

**機能**

 図面に線分を追加します。

**構文**

 LINE ['signal_name'] [width] ..
 LINE ['signal_name'] [width] [ROUND | FLAT] [curve | @radius] ..

**マウス キー** 

 *中央*クリックは、レイヤを選択します。
 *右クリック*すると、曲げスタイルが変更されます(「[SET Wire_Bend](ECD-CLI-S.html#x129)」を参照)。
 *[Shift]+右*クリックは、曲げスタイルの切り替え方向を反転します。
 線分を開始する際に *[Ctrl]+左*クリックを押すと、次の既存の線分の端点にスナップします。
 *[Ctrl]+右*クリックは、対応する曲げスタイルを切り替えます。
 線分の端点を配置する際に *[Ctrl]+左*クリックを押すと、円弧の半径が定義されます。

**関連項目**: [MITER](ECD-CLI-M.html#miter)、[SIGNAL](ECD-CLI-S.html#signal)、[ROUTE](ECD-CLI-R.html#route)、[CHANGE](ECD-CLI-C.html#change)、[NET](ECD-CLI-N.html#net)、[BUS](ECD-CLI-B.html#bus)、[DELETE](ECD-CLI-D.html#delete)、[RIPUP](ECD-CLI-R.html#ripup)、[ARC](ECD-CLI-A.html#arc)

LINE コマンドは、図面に線分(トラック)を追加するために使用されます。線分は、最初に指定した点から始まり、2 番目の点まで延びます。追加の点を指定すると、追加の線分セグメントが描画されます。同じ位置で 2 回マウス クリックすると線分が終了し、次のマウス クリックの位置で新しい線分を開始することができます。

現在アクティブな曲げスタイルに応じて、2 つの点の間に 1 つまたは 2 つの線分セグメントが描画されます。曲げスタイルはセグメント間の角度を定義し、マウスの右ボタンで変更できます([Shift]キーを押しながらマウスの右ボタンをクリックすると曲げスタイルの通過方向が反転し、[Ctrl]キーを押すと対応する曲げスタイル間で切り替わります)。

マウスの中央ボタンを押すとポップアップ メニューが表示され、線分を描画するレイヤを選択できます。

特殊なキーワードである ROUND と FLAT、および *curve* パラメータを使用して、円弧を描画できます(下記参照)。

[Ctrl]キーを押しながら線分を開始すると、新しい線分の始点が、最も近い既存の線分の座標にスナップします。これは、既存の線分がグリッドから外れている場合に特に便利です。また、現在の幅、画層、およびスタイルも既存の線分の幅、画層、スタイルに調整されます。現在の曲げスタイルが 7 ([フリーハンド])の場合、新しい線分は既存の線分からスムーズに継続されます。

### 信号名

LINE コマンドを使用して、ネット、バス、または信号のワイヤを作成することはできます(現在のレイヤによって異なる)が、お勧めしません。代わりに、[NET](ECD-CLI-N.html#x91)、[BUS](ECD-CLI-B.html#bus)、または [SIGNAL](ECD-CLI-S.html#signal) を使用することをお勧めします。signal_name パラメータは主に、生成されたデータを読み込むスクリプト ファイルで使用することを目的としています。signal_name を指定すると、この信号に後続のすべての線分がワイヤとして追加され、自動チェックは実行されません。
**この機能は、異なる信号を接続するような方法でワイヤを配置すると短絡になる可能性があるため、十分に注意して使用する必要があります。信号の signal_name パラメータを指定して LINE コマンドを使用した後で、[デザイン ルール チェック](ECD-CLI-D.html#drc)を実行します。**

### 線分とワイヤの比較

信号、ネット、またはバスの一部としての「電気的な」意味が理由で、ワイヤは線分(プレーン ワイヤ)と区別されます。従来から存在する理由で、ワイヤは Fusion ヘルプでは両方の意味で幅広く使用されています。

### 線幅(ワイヤの幅)

LINE コマンドをアクティブにした後で数値を入力すると、線分の幅が(現在の単位で)変更されます。この値は最大 200 mm (7.8740 インチ)です。線分の幅は、コマンド

CHANGE WIDTH width

を使用していつでも変更できます。

### 線種(ワイヤのスタイル)

線分は、次のいずれかの_スタイル_になります。

- Continuous

- LongDash

- ShortDash

- DashDot

線種は、[CHANGE](ECD-CLI-C.html#change) コマンドで変更できます。DRC とオートルータでは、スタイルが異なる場合でも、線分は常に「Continuous(実線)」として扱われることに注意してください。線種は主に電気図面と機械図面に使用されます。信号画層には使用しないでください。パッドに接続された信号の一部として実線以外を使用すると、明示的な DRC エラーになります。

### Top 画層、Bottom 画層、および Route 画層の信号

画層 Top、Bottom、および ROUTE2...15 の線分は、信号として扱われます。これらの画層のいずれかに既存の信号から始まる線分を描画する場合、この線分のすべてのセグメントがその信号に属します(線分の中心が正確に既存の信号ワイヤまたはパッドの中心に配置されている場合のみ)。別の信号に接続された線分セグメントを使用してこの描画操作を終了すると、Electronics から 2 つの信号を接続するかどうかを尋ねられます。Electronics では、各線分セグメントが単一のオブジェクトとして扱われることに注意してください(線分を削除する場合など)。

LINE コマンドがアクティブな場合、マウスの中央ボタンを使用して、線分を描画する画層を変更できます。

### 円弧を描画する

線分と円弧は基本的に同じオブジェクトであるため、[ARC](ECD-CLI-A.html#arc) コマンドを使用するか、必要なパラメータを LINE コマンドに追加することにより、円弧を描画できます。線分を円弧にするには、円弧の「曲率」を定義する *curve* パラメータ、または円弧の半径を定義する *@radius* パラメータ(*curve* と *radius* を区別するために必要な「@」に注意)のいずれかが必要です。*curve* の有効な値の範囲は -360 ～ +360 (+-360 は含まない)で、この値は円全体のどの部分で円弧が構成されるかを意味します。たとえば、値 90 は 90°の円弧、値 180 は半円になります。円全体をこの方法で作成することはできません(その場合は [CIRCLE](ECD-CLI-C.html#circle) コマンドを使用)。*curve* の正の値は、円弧が数学的に正の向き(反時計回り)で描画されることを意味します。*curve* が 0 の場合、円弧は直線(「曲率なし」)となり、これは実質的に線分です。*curve* パラメータと *width* パラメータを区別するために、必ず符号(「+」または「-」)を付けて指定する必要があることに注意してください。正の値である場合でも同様です。  

たとえば、コマンド

LINE (0 0) +180 (0 10);

これによって、点(0 0)から点(0 10)への半円が反時計回り方向に描画されます。*半径* を指定すると、円弧にその半径が設定されます。*curve* パラメータと同様に、*radius* にも円弧の方向を決定するための符号が必要です。たとえば、次のコマンドを実行します。

LINE (0 0) @+100 (0 200);

これによって、点(0 0)から点(0 200)への半円(半径は 100)が反時計回り方向に描画されます。始点と終点の距離が半径の2倍を超える場合は、直線が描画されます。円弧半径は、[Ctrl]キーを押しながら線分の終点を配置することで定義することもできます(通常は円弧が配置される円の中心点で行われます)。この場合、点は実際の終点として取得されず、円弧の半径の設定に使用されます。カーソルを移動して、指定した半径の円弧を配置できます([Ctrl]を押しながらマウスの右ボタンをクリックすると、円弧の方向が切り替わります)。始点から半径の 2 倍を超えた位置までカーソルを離すと、直線が描画されます。

LINE コマンドを使用して円弧を描画できるようにするために(特に生成されたスクリプト ファイルで重要)、LINE コマンドでキーワード ROUND および FLAT を使用することもできます。ただし、これらは実際の円弧にのみ適用されることに注意してください(直線は常に丸い端点を持ちます)。既定では、LINE コマンドで作成される円弧は丸い端点を持ちます。

## LOCK

**機能**:

 ボード内のパーツの位置と方向をロックします。

**構文**:

 LOCK ..
 LOCK name ..LOCK - LOCK *; LOCK -*;

**マウス キー**:

 *[Ctrl]+右*クリックは、コマンドをグループに適用します。
 *[Shift]+左*クリックは、ロック操作を反転します(オブジェクトを「ロック解除」)。
 *[Ctrl]+[Shift]+右*クリックは、ロック プロパティを持つグループ内のすべてのオブジェクトを「ロック解除」します。
