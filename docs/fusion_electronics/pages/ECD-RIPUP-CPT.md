# オートルータ リップアップと再試行の試行回数

オートルータの構造により、リップアップと再試行メカニズムに影響を与えるために使用できるパラメータがいくつかあります。所要時間と配線結果とを適切に調整できます。必要に応じて *mnRipupLevel、mnRipupSteps、*mnRipupTotal の値を変更する場合は、細心の注意を払ってください。

一般に、パラメータ値が高いと、リップアップは増えますが、計算時間は長くなります。

パラメータの意味を理解するためには、ルータの動作を知ると役に立ちます。トラックは、他のパスが見つからなくなるまで順次配線されます。他のパスが見つからなくなると、新規トラックを配線するために、既に配線したトラックがその最大数まで削除されます(最大数は *mnRipupLevel* で定義されています)。たとえば、既に 8 個のトラックがある場合、新規トラックを配線できるのは mnRipupLevel が 8 以上の場合のみです。

新規トラックを配線した後、ルータは削除したすべてのトラックを再配線しようとします。これらのトラックのいずれかを再配線するために、場合によっては新しいリップアップ シーケンスを開始する必要があります。その場合、配線できなかったトラックがあるため、ルータはプロセス全体を開始した位置から離れた 2 つのリップアップ シーケンスとなります。再配線できない削除済みの各トラックが、新しいリップアップ シーケンスを開始します。このようなシーケンスの最大数は、*mnRipupSteps* パラメータで定義します。

*mnRipupTotal* パラメータでは、同時に削除できるトラックの数を定義します。場合によっては、この値を超過することがあります。

これらの値のいずれかを超過すると、リップアップ プロセスが中断し、配線できなかった最初のトラックで有効であったステータスが再設定されます。このトラックは配線できないトラックとみなされ、次のトラックに進みます。
