# プライベート ライブラリをハブに読み込む

Library.io から現在のハブにプライベート ライブラリを読み込むことで、すべてのチーム メンバーがコンポーネントにアクセスできるようになります。こうすることで、コンポーネントの編集や、2D および 3D PCB デザインの同期を行う際の権限の問題を回避できます。

## 前提条件

- [電子デザイン ライブラリ マネージャを開く](ECD-OPEN-LIBMGR-TSK.htm)

## 手順

- **電子デザイン ライブラリ マネージャ**で、 **[プライベート]** タブをクリックします。

- 読み込むライブラリを右クリックします。

- ダウンロードのオプションが表示された場合:

**[ダウンロード]** ![ライブラリのダウンロード アイコン](../images/icon/ecd/lbr-download.png) を選択して、ライブラリをダウンロードします。

- ダウンロードが完了したら、ライブラリをもう一度右クリックして、読み込みアクションを表示します。

- 使用可能なアクションを選択します。

**Fusion ハブにインポート** ![ライブラリの読み込みアイコン](../images/icon/ecd/lbr-import.png): ライブラリがまだ読み込まれていない場合。

- **新規ハブ ライブラリとしてインポート** ![ライブラリの読み込みアイコン](../images/icon/ecd/lbr-import.png): ライブラリが既に読み込まれており、2 番目のコピーを作成する必要がある場合。

## 重要事項

- **ハブ ライブラリが見つからない:** ライブラリ マネージャでハブ ライブラリが見つからない場合、 **[読み込み済み]** 列に間違った情報が表示される可能性があります。重複して読み込まれることを防ぐために、[ハブをスキャン](ECAD-ADD-MISSING-HUB-LBRS-TSK.htm)してそれらのライブラリを見つけます。

- **ローカル ディスク ライブラリ:** **[読み込み済み]** 列は、Library.io ライブラリにのみ適用されます。

- **プライベート パッケージ:** Library.io に保存されているプライベート パッケージは、自動的には読み込まれません。[これらのパッケージは個別に読み込む](ECAD-IMPORT-PRIVATE-PCKGS-TSK.htm)必要があります。

## 次のステップ

- **既存のデザインを更新する:** プライベート ライブラリのコンポーネントを使用するデザインがある場合は、[ライブラリを入れ替える]機能を使用して、デザインでハブ ライブラリのコンポーネントのみが使用されるようにします。

- **プライベート パッケージを読み込む:** ライブラリで使用している場合は、[プライベート パッケージを読み込みます](ECAD-IMPORT-PRIVATE-PCKGS-TSK.htm)。
