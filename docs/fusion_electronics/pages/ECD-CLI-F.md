## FANOUT

**関数**

 デバイスおよび信号からファンアウトを作成します。

**構文**

 FANOUT DEVICE device-name [direction]
  FANOUT SIGNAL signal-name [direction]

**マウス キー**

 *左*クリックは、ファンアウトするデバイスまたは信号を選択します。

**関連項目**: [ROUTE](ECD-CLI-R.html#route)、[CHANGE](ECD-CLI-C.html#change)

FANOUT コマンドは、配線済みワイヤを作成し、選択したデバイスまたは信号のパッドからビアをアタッチします。表面実装型デバイス(SMD)またはボールグリッド アレイ(BGA)デバイスのみがファンアウトされます。貫通穴パッドを持つデバイスは無視されます。

WIDTH パラメータはファンアウトで配線されるワイヤの幅を定義し、ROUTE コマンドで使用されるパラメータと同じパラメータです。幅は、次のコマンドを使用して変更できます。

CHANGE WIDTH width;

ここで、*width* は現在の単位の目的の値です。幅がデザイン ルールの最小幅より小さい場合、デザイン ルール違反を避けるために最小幅が使用されます。

ファンアウトに使用されるビアは、[ドリル]と[直径]の両方のパラメータ ツールバー設定によって定義されます。width パラメータと同様に、ドリル サイズがデザイン ルールの最小ドリルより小さい場合、最小サイズが使用されます。ドリルは、次のコマンドを使用して変更できます。

CHANGE DRILL drill-size;

パッドからビアが配置される場所までの距離は、同じ信号のクリアランスを示すデザイン ルール[SMD からビア]によってコントロールすることができます。[DESIGN RULES](ECD-CLI-D.html#x181) (DRC)コマンドを参照してください。

ファンアウト ワイヤまたはファンアウト ビアがデザイン ルール違反を引き起こす場合、パッドからの特定のファンアウトは実行されません。このような場合は、幅およびドリル サイズのパラメータ、およびデザイン ルールの最小値([ネット クラス]を含む)を参照して、幅またはドリルの調整が必要かどうかを確認します。これらの値を変更できない場合は、ファンアウト代替のファンアウト方向を使用してみると、このような場合に役立つ場合があります。

パラメータのツールバー ボタン[違反を許可]がオンになっている場合、ビアが隣接ファンアウトに対してデザイン ルール違反を引き起こす場合でも、FANOUT コマンドを使用してビア ファンアウトを作成することができます。このオプションを使用すると、特定のビアによって発生する違反を確認できるため、特定のビアがファンアウトされない理由を判断する際に役立ちます。このオプションの既定値はオフで、違反は許可されません。

### 方向

デバイスを基準にしたファンアウトの方向は、ツールバーのファンアウト方向パラメータを使用するか、次のオプションのコマンド ライン パラメータを使用することで、コントロールすることができます。

| 

ﾘｰﾄﾞｱｳﾄ | 

 | 

ファンアウトはデバイスの中心から外側に遠ざかります | 

 |

| 

ﾁ | 

 | 

ファンアウトはデバイスの中心に向かって内側に近づきます | 

 |

| 

回転方向反転 | 

 | 

ファンアウトは、隣接するパッドに対して OUT と IN が交互に切り替わります | 

 |

### 例

FANOUT DEVICE U3;
FANOUT DEVICE U4 OUT;
FANOUT SIGNAL GND ALTERNATE;
FANOUT SIGNAL +5;

## FLIP

**機能**

 オブジェクトとグループを反転します。

**構文**

 FLIP [ルート モード] ..
 FLIP [ルート モード] 名前..

**マウス キー**

 *[Ctrl]+右*クリックは、グループを反転します。

**関連項目**: [ROTATE](ECD-CLI-R.html#rotate)、[LOCK](ECD-CLI-L.html#lock)、[TEXT](ECD-CLI-T.html#text)、[MIRROR](ECD-CLI-M.html#mirror)

FLIP コマンドは、Y 軸を中心にオブジェクトをミラー化し、ボードの反対側に反転させます。

パーツ、パッド、SMD、およびピンを名前で選択することができます。これは、特に、オブジェクトが現在表示されているウィンドウ領域の外にある場合に便利です。

パーツの属性は、パーツ名と属性名を連結して入力することによって選択できます(例: R5>VALUE)。

FLIP コマンドで使用するフットプリントを選択すると、外側レイヤーで接続されているワイヤも反転します(短絡に注意)。

内側レイヤー(2...15)にある穴やビアなどのオブジェクトは、反転してもレイヤーは変更されないことに注意してください。

パーツが[ロックされている](ECD-CLI-L.html#lock)場合、または接続されているいずれかのパッドが許容領域の外側に延長される場合、そのパーツを反転することはできません。

### ルート モード

接続されているワイヤの配線動作は、ツールバーのルート モード パラメータ、または次のコマンド ライン オプションを使用してコントロールできます。

| 

REROUTE | 

 | 

接続されているワイヤが再配線されます | 

 |

| 

DISCONNECT | 

 | 

接続されているワイヤが切断され、所定の位置に残ります | 

 |

| 

UNROUTE | 

 | 

接続されているワイヤが削除されます | 

 |

### 文字を反転する

プリント基板の下側の文字は、ボードの下側を見たときに読めるように自動的にミラー化されます。回路図のミラー化された文字は、原点の反対側に印刷されますが、通常通り読むことができます。

### 例

FLIP U3;
FLIP UNROUTE U4;

## FRAME

**関数**

 図面にフレームを追加します。

**構文**

 FRAME [ columns [ rows ] ] [ borders ]  

**マウス キー**

 *中央*クリックは、レイヤを選択します。

**関連項目**: [LABEL](ECD-CLI-L.html#label)

FRAME コマンドは、番号付けされた列と行を持つフレームを描画します。2 つの点で、フレームの 2 つの対角コーナーを定義します。マウスの中央ボタンを押すと、フレームを追加する画層が変更されます。

columns パラメータは、フレーム内の列数を定義します。最大 127 列まで指定できます。既定では、左から右の順に、列の番号が付けられます。columns 値が負の場合、右から左の順に番号が付けられます。

rows パラメータは、フレーム内の行数を定義します。最大 26 行まで指定できます。行は、上から下の順に、「A」で始まる文字でマークされます。rows 値が負の場合、下から上の順にマークされます。rows を指定する場合は、その前に columns を追加する必要があります。

borders パラメータを指定すると、番号または文字が表示される境界をフレームのどの側に表示するかを定義できます。このパラメータの有効なオプションは、Left、Top、Right、および Bottom です。既定では、フレームの 4 つの側すべてに境界があります。これらのオプションのいずれかが指定されている場合、要求された側にのみ境界が表示されます。特殊なオプション None および All を使用すると、「境界なし」および「すべての側」をマークすることができます。

同じ図面に複数のフレームを描画することはできますが、パーツとネットの位置の計算には最初のフレームのみが使用されます。これらの位置を使用して(たとえばユーザ言語プログラムで)、パーツおよびそれぞれのフレーム内での位置のリストを生成することができます。また、これらの位置はラベルの相互参照を自動的に生成するために内部で使用されます。

フレーム オブジェクトには特別な性質があるため、独自の回転はなく、[ROTATE](ECD-CLI-R.html#rotate) コマンドでは回転されません。

フレームをボードまたは回路図に直接描画することはできますが、より一般的な使用方法として、特殊な記号またはフットプリントの図面(通常は表題欄も含む)を作成すると、その後、これをすべての図面で使用することができます。EAGLE に付属の「frames」ライブラリには、いくつかの図面フレームが含まれています。

### 例

FRAME 10 5 TOP LEFT  

10 列(左から右の順に番号付け)と 5 行(上から下の順に「A」～「E」でマーク)を持つフレームを描画します。列インジケータと行インジケータは上境界と左境界にのみ描画されます。

## FUSIONSYNC

**機能**

 ボードを Fusion PCB と同期します。

**構文**

 FUSIONSYNC  

FUSIONSYNC コマンドは、ボードを Fusion PCB モデルと同期するために使用されます。ユーザは、PCB の 3D 表現を簡単に作成できます。ボードを EAGLE で編集する際に、このコマンドを使用すると、変更内容を Fusion PCB モデルにプッシュすることができます。ボードを Fusion で編集する際に、このコマンドを使用すると、これらの変更内容をプルし、EAGLE ボード ファイルに統合することができます。Fusion PCB モデルへのリンクがボードに保存されます。このコマンドを使用すると、ボードを別の Fusion PCB モデルにリンクすることができます。

## FUSIONTEAM

**関数**

 回路図ファイル、ボード ファイル、および製造データ ファイルを Fusion Team にパブリッシュします。

**構文**

 FUSIONTEAM

回路図ファイルおよびボード ファイルを Fusion Team にアップロードすることができます。レイアウト エディタでコマンドをアクティブにするには、[Fusion Team]タブをクリックするか、コマンド ラインにコマンドを入力します。製造データを処理するには、CAM ジョブ ファイルを選択します。製造データは、自動的に ZIP 圧縮アーカイブとして Fusion Team にアップロードされます。[Fusion Team]タブは、[オプション]/[ユーザ インタフェース]メニューの[Fusion 360]タブと一緒にアクティブ(非アクティブ)にできます。
