## EDIT

**関数**

 編集する既存の図面をロードするか、新しい図面を作成します。

**構文**

 EDIT name
  EDIT name.ext
  EDIT .ext
  EDIT .sX [ .sY ]
  EDIT name.mod
  EDIT name.m2
  EDIT .m3
  EDIT modX.m3 modY.m1
  EDIT;

**関連項目**: [OPEN](ECD-CLI-O.html#open)、[CLOSE](ECD-CLI-C.html#close)、[BOARD](ECD-CLI-B.html#board)、[MODULE](ECD-CLI-M.html#module)

EDIT コマンドを使用すると、図面が開きます。開いているライブラリの場合、パッケージ、シンボル、またはコンポーネントを編集用に開きます。

EDIT コマンドは、回路図内のモジュールを作成または編集する場合にも使用します。

名前にはワイルドカードを使用することができます(*.brd など)。

パラメータを指定せずに EDIT コマンドを実行すると、ファイル ダイアログ(ボード モードまたは回路図モード)またはポップアップ メニュー(ライブラリ モード)が表示され、そこからファイルまたはオブジェクトを選択できます。

回路図から同じ名前のボードに切り替えるには、コマンド

EDIT .brd

を使用できます。同様に、ボードから回路図に切り替えるには、次のコマンドを使用します。

EDIT .sch

ライブラリ エディタからボードまたは回路図に切り替えることもできます。回路図の別のシートを編集するには、次のコマンド

EDIT .sX

(X はシート番号)、またはエディタ ウィンドウのアクション ツールバーのコンボ ボックスを使用します。指定されたシート番号が存在しない場合は、新しいシートが作成されます。シートのサムネイル プレビューのアイコンをクリックし、シートを切り替えることもできます。サムネイル プレビューでドラッグ アンド ドロップして、シートを並べ替えることができます。

記号、コンポーネント、またはフットプリントは、ライブラリが最初に OPEN コマンドを使用して開かれた場合にのみ編集できます。これを行うには、コンポーネントの名前または URN を使用します。

### ファイルをロードするフォルダ

EDIT は、プロジェクト フォルダからファイルをロードします。

## ERC

**関数**

 電気的ルール チェック。

**構文**

 ERC

**関連項目**: [DRC](ECD-CLI-D.html#drc)、[ERRORS](#errors)

 ERC は、電気的エラーについて回路図をテストするために使用されます。チェックの結果は、[[エラー]](#x62)ダイアログに表示されます。

### 整合性チェック

ERC コマンドは、ERC を開始する前にボード ファイルがロードされている場合、回路図と対応するボードとの間の整合性チェックも実行します。このチェックの結果、ファイルが一貫しているかどうかに応じて、自動 Forward&Back Annotation がオンまたはオフになります。ERC は、回路図の暗黙的電源ピンおよび供給ピンと、ボードの実際の信号接続との間の矛盾を検出することに注意してください。このような不整合は、BOARD コマンドを使用してボードを作成した後で供給ピン設定を変更すると発生する可能性があります。電源ピンは「暗黙的に」接続されているだけなので、これらの変更に対して常に Forward Annotation が実行されるとは限りません。
このようなエラーが検出された場合にも Forward&Back Annotation が実行されますが、供給ピン設定を確認する必要があります。

## ERRORS

**関数**

 ERC コマンドまたは DRC コマンドによって検出されたエラーを表示します。

**構文**

 ERRORS
  ERRORS CLEAR

**関連項目**: [ERC](#erc)、[DRC](ECD-CLI-D.html#drc)

電気的ルール チェック(ERC)またはデザイン ルール チェック(DRC)によって検出されたエラーを表示するには、ERRORS を使用します。選択すると、ウィンドウが開き、すべてのエラーが表示されます。ロードされた図面に対して ERC または DRC が実行されていない場合、最初にそれぞれのチェックが開始されます。

[エラー]ダイアログの一覧表示には、*一貫性エラー、*エラー*、*警告 および _承認済み_メッセージを含む最大 4 つのセクションがあります。

エントリを選択すると、エディタ ウィンドウの画面の左上隅からエラー箇所まで、長方形と線でエラーが示されます。

エントリをダブルクリックすると、エラーがある領域に図面が中心合わせされます。これは、[中心]チェックボックスをオンにすると自動的に行われます。

### メッセージを処理済みとしてマークする

_[処理済み]_ボタンはメッセージを処理済みとしてマークします。このメッセージは引き続きリストに表示されますが、エディタ ウィンドウのエラー インジケータは表示されなくなります(そのリスト エントリが選択されている場合は除く)。これは、関連する問題が修正された後で、メッセージを「完了」としてマークするために使用できます。再度チェックを実行する必要はありません。次の ERC または DRC の後、メッセージは表示されなくなるか、問題が解決されていない場合は再度未処理としてマークされます。

### メッセージを承認する

エラーまたは警告を修正できないが、重要でないと思われる場合は(ユーザが決定する必要があります)、*[承認]_ボタンを押して*[承認済み]_セクションに移動できます。このセクションのメッセージについては、エディタ ウィンドウにエラー インジケータは描画されず(そのリスト エントリが選択されている場合は除きます)、暗黙的に「処理済み」としてマークされます。次の ERC または DRC の後でこれらのメッセージのいずれかが適用されなくなった場合は、削除されます。承認されたすべてのメッセージが図面ファイルに保存されるため、ユーザによって明示的に承認されたメッセージがドキュメント化されます。一貫性エラーは承認できないため、Forward&Back Annotation をアクティブにするためには常に修正する必要があることに注意してください。

### リストをクリアする

_[すべてクリア]_ボタンをクリックすると、承認済みメッセージを除くすべてのエントリがリストから削除されます。これは、エディタ ウィンドウのエラー インジケータを除去するために使用できます。次の ERC/DRC では、まだ問題が解決していない場合は、エラーメッセージが再表示されます。このリストは、次のコマンドを入力してクリアすることもできます

ERRORS CLEAR

## EXPORT

**関数**

 データ ファイルの生成。

**構文**

 EXPORT SCRIPT filename;
  EXPORT NETLIST filename;
  EXPORT SPICENETLIST filename;
  EXPORT NETSCRIPT filename;
  EXPORT PARTLIST filename;
  EXPORT PINLIST filename;
  EXPORT DIRECTORY filename;
  EXPORT IMAGE filename|CLIPBOARD [MONOCHROME|WINDOW] resolution;

**関連項目**: [SCRIPT](ECD-CLI-S.html#script)、[RUN](ECD-CLI-R.html#run)

EXPORT コマンドは、EAGLE から他のプログラムへのデータの転送や現在の図面からのイメージ ファイルの生成などに使用できる、ASCII テキスト ファイルをユーザに提供するために使用されます。

既定では、出力ファイルは**[プロジェクト]**フォルダに書き込まれます。

このコマンドは、次の出力ファイルを生成します。

#### SCRIPT

以前に OPEN コマンドで開いたライブラリはスクリプト ファイルとして出力されます。ライブラリがエクスポートされていて、再度 SCRIPT コマンドを使用してインポートする場合、重複(同じ記号が複数回定義されるなど)を避けるために新しいライブラリを開いておく必要があります。スクリプト ファイルの読み込み速度を上げるには、最初に

Set Undo_Log Off;

を使用して元に戻すログを無効にしてください。

#### NETLIST

ロードされた回路図またはボードのネットリストを生成します。要素に接続されているネットのみがリストされます。

#### SPICENETLIST

ロードされた回路図の SPICE 形式のネットリストを生成します。要素に接続されているネットのみがリストされます。

#### NETSCRIPT

ロードされた回路図のネットリストをスクリプト ファイルの形式で生成します。このファイルは、要素が既に配置されているボードや、前に配線されていたトラックが DELETE SIGNALS で削除されているボードに、新しいネットリストまたは変更されたネットリストを読み込むために使用されます。このようなスクリプトをボードに読み込んでいる間は、このボードと一致する回路図をロードできないことに注意してください。

#### PARTLIST

回路図またはボードのコンポーネント リストを生成します。ピンおよびパッドを持つ要素のみが含まれます。

#### PINLIST

パッドとピンのリストを生成します。ピンの方向と、ピンに接続されているネットの名前が含まれます。

#### DIRECTORY

現在開いているライブラリのフォルダが一覧表示されます。

#### IMAGE

_イメージ_をエクスポートすると、指定したファイル名拡張子に対応する形式のイメージ ファイルが生成されます。以下のイメージ フォーマットを使用できます。

| 

.bmp | 

 | 

Windows ビットマップ ファイル | 

 |

| 

.png | 

 | 

ポータブル ネットワーク グラフィック ファイル | 

 |

| 

.pbm | 

 | 

ポータブル ビットマップ ファイル | 

 |

| 

.pgm | 

 | 

ポータブル グレースケール ビットマップ ファイル | 

 |

| 

.ppm | 

 | 

ポータブル ピクセルマップ ファイル | 

 |

| 

.tif | 

 | 

TIFF ファイル | 

 |

|

 |

| 

.xpm | 

 | 

X ピクスマップ ファイル | 

 |

*resolution* パラメータは、イメージの解像度(「dpi」単位)を定義します。

*filename* が特別な名前 CLIPBOARD (大文字と小文字は区別されません)の場合、イメージはシステムのクリップボードにコピーされます。

オプションのキーワード *MONOCHROME* を指定すると、白黒のイメージが作成されます。

オプションのキーワード *WINDOW* を指定すると、エディタ ウィンドウで現在表示されている領域のイメージが作成されます。このキーワードを指定しないと、イメージには図面全体が含まれます。

### その他の形式

DXF や Hyperlynx などのその他の多くの形式を ULP でエクスポートできます。これらは、[RUN](ECD-CLI-R.html#x127) コマンドを使用してコマンド ラインから開始することができます。[ファイル]/[エクスポート]では、これらの多数の形式でのエクスポートを使用できます。

## EXPORTSTEP

**関数**

 ボードを STEP ファイルにエクスポートします。

**構文**

 EXPORTSTEP

EXPORTSTEP コマンドは、コマンド ラインで使用することも、[レイアウト エディタ]ウィンドウの[エクスポート]/[STEP]メニューから使用することもできます。EAGLE では、Fusion Sync のクラウド サービスを使用して変換を行います。STEP ファイルには、コントロール パネルのホーム タブからアクセスできます。このファイルは、[最近生成した 3D ファイル]セクションに表示されます。リスト内のエントリをクリックすると、関連するステップ ファイルを含むファイル マネージャ フォルダが開きます。
