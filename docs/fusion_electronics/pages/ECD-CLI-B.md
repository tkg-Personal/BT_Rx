## BOARD

**機能**:

 回路図をボードに変換します。

**構文** 

 BOARD [ grid ]

**関連項目**: [EDIT](ECD-CLI-E.html#edit) 

回路図図面をボードに変換するには、BOARD コマンドを使用します。ボードが既に存在する場合は、ボード ウィンドウにロードされます。ボードが存在しない場合は、新しいボードを作成するかどうかを確認するメッセージが表示されます。グリッドが指定されている場合、ボード上のパーツは次のように指定されたラスターに配置されます。

BOARD 5mm

この場合、パーツは 5 mm のラスター(既定は 50 mil)に配置されます。数値は単位で指定する必要があり、最大値は 10 mm です。BOARD コマンドは、既存のボードファイルを上書きすることはありません。同じ名前のファイルが既に存在する場合に新しいボード ファイルを作成するには、最初にそのファイルを[削除](ECD-CLI-R.htm)する必要があります。

### 回路図からボードを作成する

ボードを初めて編集する際に、プログラムが同じフォルダに同じ名前の回路図があるかどうかをチェックし、その回路図からボードを作成するかどうかをユーザが選択します。
回路図ウィンドウが開いているときにボードを作成する場合は、エディタ ウィンドウのコマンド ラインで

edit .brd

と入力します。回路図ファイル(name.sch)のすべての関連データがボード ファイル(name.brd)に変換されます。新しいボードが、サイズが 160 x 100 mm (Light エディション: 100 x 80 mm)の空のカードとして自動的にロードされます。すべてのパッケージと接続がボードの左側に表示されます。供給ピンは既に接続されています([PIN](ECD-CLI-P.html#pin) を参照)。

既定で生成されるものとは異なるボードのアウトラインが必要な場合は、それぞれの線分を削除してから、[LINE](ECD-CLI-L.html#line) コマンドを使用して独自のアウトラインを *Dimension* レイヤに描画します。これらの線分の推奨幅は 0 です。

次の場合は、ボード ファイルを生成できません。

- パッケージが定義されていないデバイスの回路図にゲートが存在する場合(エラー メッセージ: 「デバイス名にパッケージがありません」)。例外: 方向が「Sup」(供給シンボル)のピンのみが存在する場合

- 一部のピンがフットプリントの関連パッドに割り当てられていない場合。例外: ピンのないデバイス(フレームなど)

## BREAKOUTBUS

**関数** 

 バスからメンバー ネットをラベル付きの新しいネットとしてブレークアウトします。 

 **構文**

  BREAKOUTBUS ..

BREAKOUTBUS コマンドには、バスを右クリックした後でコンテキスト メニューからアクセスします。ブレークアウトのタイプは、メニューで[すべてのネット]、[選択したネット]、[単一ネット]の中から選択できます。[選択したネット]オプションを選択すると、ブレークアウトするネットを選択できるダイアログが表示されます。[単一ネット]は、バスのすべてのメンバーが表示されるサブメニューであり、単一ネットを選択できます。[すべてのネット]は、バスのすべてのメンバーをブレークアウトします。

ブレークアウトのタイプを選択すると、バスに沿って 1 グリッドの間隔でラベルを持つ新しいネットが作成され、ユーザがクリックすると新しいネットの配置が終了します。ネットを配置するときに[Shift]を押したままにすると、新しいネットの形状がフリーズされ、バスに沿った移動のみが許可されます。

## BUS

**関数**

 回路図にバスを描画します。

**構文**

 BUS [? | {PREDEFINED_BUS_NAME} | bus_spec] [curve | @radius] ..
 BUS {PREDEFINED_BUS_NAME=bus_spec}

**マウス キー** 

 *右クリック*すると、ワイヤ曲げスタイルが変更されます(「[SET Wire_Bend](ECD-CLI-S.html#x129)」を参照)。|  *[Shift]+右クリック*は、曲げスタイルの切り替え方向を反転します。
 *[Ctrl]+右*クリックは、対応する曲げスタイルを切り替えます。 

**関連項目**: [INFO](ECD-CLI-I.html#info)、[BREAKOUTBUS](#breakoutbus)、[NET](ECD-CLI-N.html#net)、[NAME](ECD-CLI-N.html#name)、[SET](ECD-CLI-S.html#set)

バスはネットの集まりです。バスには仕様があり、1 つまたは複数のバス線セグメントとして描画されます。バス仕様には(厳密に一意の)エイリアスを含めることができます。このエイリアスは、バスを識別するために使用される短い名前です。BUS コマンドは、回路図の Bus 画層にバスを描画するために使用されます。

EAGLE には、再利用可能なバス仕様を「定義済みバス」(PDB)として保存する機能があります。PDB は eaglerc ファイルに保存され、各プロジェクトにわたって使用でき、他と共有することもできます(例: SPI:MISO,MOSI,CLK,CS)。BUS コマンドの後に中括弧で囲んだ PDB を含めると、そのバスは描画時にこの仕様を採用します。この方法で使用するには、PDB 名が有効で、eaglerc ファイルに存在している必要があります。

BUS {PREDEFINED_BUS_NAME}

BUS ツールバーのコマンド ボタンは、既存の PDB にすばやくアクセスできるコンテキスト メニューをサポートしています。このメニューから選択することは、上記のコマンド形式を使用することと同じです。

次の形式を使用して、コマンド ラインから新しい PDB を作成することもできます。PDB 名は一意である必要があり、bus_spec はこのセクションで説明されている有効なバス仕様である必要があります。PDB は eaglerc に保存され、すぐに使用できます。

BUS {PREDEFINED_BUS_NAME=bus_spec}

コマンド ラインで BUS ? と入力(またはコマンド ツールバーで[バス]ボタンのコンテキスト メニューから[新規]を選択)すると、[定義済みバス]ダイアログが開き、PDB を作成および管理できます。

BUS ?

bus_spec の形式は次のとおりです。

ALIAS:partbus,partbus,..

ALIAS は任意の名前です。partbus は、単純なネット名、または次の形式の名前の範囲です。

NetName[LowestIndex..HighestIndex]

条件 0 <= LowestIndex <= HighestIndex <= 511 を満たす必要があります。

名前が範囲とともに使用される場合、NetName が数字で終わってはなりません。これは、どの桁が名前に属し、どの桁が範囲に属しているかがわからなくなるためです。

既に別のバス ワイヤが存在する点にバス ワイヤを配置すると、現在のバス ワイヤはその点で終了します。この機能を無効にするには、「SET AUTO_END_NET OFF;」を使用するか、[オプション]/[設定]/[その他]/[ネットとバスを自動終了]を選択解除します。

パラメータ *curve* または *@radius* が指定されている場合、バスの一部として円弧を描画できます(詳細は [LINE](ECD-CLI-L.html#line) コマンドを参照)。

### バス仕様の例

A[0..15]
RESET
DB[0..7],A[3..4]
ATBUS:A[0..31],B[0..31],RESET,CLOCK,IOSEL[0..1]

バス仕様が指定されていない場合、B$1 という形式の仕様が自動的に割り当てられます。これは、NAME または INFO コマンドを使用していつでも変更できます。たとえば、次の式を使用して、バスで使用する線幅を定義できます。

SET Bus_Wire_Width 40;

40 mil に定義できます(既定: 30 mil)。

### 反転信号

反転信号(「active low」)の名前は、次のように前に感嘆符(「!」)が付いている場合に上線付きで表示できます。

  ATBUS:A[0..31],B[0..31],!RESET,CLOCK,IOSEL[0..1]

これは、次のような結果になります。

                          _____
  ATBUS:A[0..31],B[0..31],RESET,CLOCK,IOSEL[0..1]

詳細については、[TEXT](ECD-CLI-T.html#text) コマンドの説明を参照してください。

### バス ブレークアウト

バスは、[BREAKOUTBUS](#x44) コマンドを使用して、新しいラベル付きネットで自動的にブレークアウトすることができます。このコマンドは、コマンド ライン、またはバス オブジェクトの右クリック コンテキスト メニューで使用できます。
