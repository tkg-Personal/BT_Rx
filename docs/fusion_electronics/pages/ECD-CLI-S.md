## SCRIPT

**関数**

 コマンド ファイルを実行します。

**構文**

 SCRIPT file_name;

**関連項目**: [SET](#set)、[MENU](ECD-CLI-M.html#menu)、[ASSIGN](ECD-CLI-A.html#assign)、[EXPORT](ECD-CLI-E.html#export)、[RUN](ECD-CLI-R.html#run)

SCRIPT コマンドは、スクリプト ファイルに保存されているコマンドのシーケンスを実行するために使用されます。キーボードで SCRIPT と入力し、「file_name」に拡張子がない場合は、自動的に「.scr」が使用されます。

### 例

| 

SCRIPT nofill | 

 | 

nofill.scr を実行します。 | 

 |

| 

SCRIPT myscr. | 

 | 

myscr (接尾表記なし)を実行します。 | 

 |

| 

SCRIPT myscr.old | 

 | 

myscr.old を実行します。 | 

 |

スクリプト ファイルのさまざまな可能性については、EXPORT コマンドを参照してください。

マウスで SCRIPT コマンドを選択すると、ポップアップ メニューに拡張子が「.scr」のファイルがすべて表示されるので、それらのファイルを選択して実行することができます。

SCRIPT コマンドでは、目的に応じてプログラムをカスタマイズすることができます。たとえば、次のように調整を行います。

- コマンド メニューの変更

- キーの割り当て

- プリント基板形状のロード

- 色の変更

SCRIPT ファイルには、構文ルールに従って EAGLE コマンドが含まれています。「#」で始まる行はコメントです。

### 行の継続

SCRIPT ファイルには、構文ルールに従って、各行に 1 つまたは複数のコマンドが含まれています。コマンド ラインの最後に「\」という文字を含めると、次の行の最初の単語がコマンドとして解釈されなくなります。この機能により、多くの場合、アポストロフィを回避することができます。

### 既定パラメータの設定

スクリプト ファイル *eagle.scr* (プロジェクト フォルダまたはスクリプト パスに存在する場合)は、新しい図面がエディタ ウィンドウにロードされるたびに(またはライブラリで図面タイプが変更されたときに)実行されます。

### スクリプト ラベル

既定のスクリプト ファイル *eagle.scr* では、formEDITOR のラベルが使用されます。
ここで、EDITOR は SCH、BRD、LBR、DEV、PAC、SYM のいずれかです。これにより、エディタでは適切なセクションのみが実行されるようになります。たとえば、新しいボードを開くと、「BRD:」で始まるセクションのみが(次のラベルがある場合はそのラベルまで)実行されます。また、[MENU](ECD-CLI-M.html#menu) コマンドを使用して、エディタ固有のメニューを作成することもできます。ラベルは、最初の行に配置する必要があります。

### ライブラリ エディタでのスクリプト ファイルの実行

ライブラリ エディタが既にロードされている場合にのみ、すべてのレイヤが認識されます。

## 選択フィルタ

SET SELECTTYPES arg1 arg2 …

このコマンドを使用すると、選択フィルタ内の選択をオブジェクト タイプ別��制御できます。キーワード ALL と NONE は、それぞれすべてのオブジェクト タイプを有効または無効にします。「-」記号を使用すると、特定のオブジェクト タイプを無効にすることができます。

例:

SET SELECTTYPES ALL -Via

このコマンドは、選択フィルタ内のすべてのオブジェクト タイプを有効にし、ビア オブジェクト タイプを無効にします。

SET SELECTLAYERS arg1 arg2 …

このコマンドは、オブジェクトがどのレイヤ上にあるかに基づいて、選択フィルタでの選択を制御します。キーワード ALL と NONE は、それぞれすべてのレイヤの選択を有効または無効にします。「-」記号を使用すると、特定のレイヤを無効にすることができます。

例:

SET SELECTLAYERS None Top

このコマンドは、すべてのレイヤを無効にしてから上部レイヤを有効にして、上部レイヤ上のオブジェクトだけを選択できるようにします。

### 色

黒、白、カラーの背景のそれぞれに対応した 3 つの_パレット_があります。各パレットには 64 色のエントリがあり、任意の ARGB 値に設定することができます。パレット エントリ番号 0 は、背景色として使用されます(「白」パレットでは、パレットが印刷にも使用されるため、このエントリは変更できません。つまり、背景は常に白になります)。カラー パレットは、[オプション]/[設定...]/[色]のダイアログを使用するか、次のコマンドを使用して変更できます。

SET PALETTE _index_ _argb_

ここで、*index* は 0 ～ 63 の数値で、*argb* は色のアルファ、赤、緑、青の要素を定義する 16 進数値です(0xFFFFFF00 とすると、明るい黄色になります)。アルファ要素は、カラーの「不透明」度を定義します。値を 0x00 にすると完全に透明(つまり、非表示)になり、0xFF にすると完全に不透明になります。背景色のアルファ要素は常に 0xFF です。ARGB 値は「0x」で始める必要があります。それ以外の場合は、十進数とみなされます。次のコマンドを使用すると

SET PALETTE BLACK|WHITE|COLORED

黒、白、またはカラーの背景パレットにそれぞれ切り替えることができます。このコマンドを実行してもウィンドウは自動的に更新されません。そのためこの後 WINDOW;コマンドを実行する必要があります。既定では、パレットエントリ 0から 15 のみが使用され、以下の色が含まれています。

パレット エントリは、「標準」色と「ハイライト」色にグループ化されています。常に 8 つの「標準」色があり、その後に対応する 8 つの「ハイライト」色が続きます。0 ～ 7 の色が「通常」色の値で、8 ～ 15 の色が「ハイライト」色の値です。16 ～ 23 がまた別の 8 個の「標準」色の値で、24 ～ 31 が「ハイライト」色の値というようになります。「ハイライト」色は、オブジェクトを表示するために使用します。たとえば、SHOW コマンドで使用します。

色は色番号に従って一覧表示され、色番号は色名の代わりに使用することができます。次のように、色を指定するために使用します。

| 

0 |

### 自動確認

Electronics では、情報メッセージまたは警告メッセージで確認を求められたり、今後の操作方法を決定するように求められる場合があります。(スクリプト ファイルのような)自動化プロセスでは、これが不要になることもあります。コマンド

SET CONFIRM YES

は、ユーザが肯定ボタン([OK]または[はい])をクリックした場合と同様に、すべてのメッセージ ダイアログが自動的に確認されます。ダイアログ自体はユーザに表示されません。同様に、次のコマンドは、

SET CONFIRM NO

否定ボタン([いいえ])ボタンが存在する場合には、それをクリックした場合と同様に動作します。それ以外の場合は、単にダイアログを確認します。追加のオプションでダイアログを確認するためのキーワードとして OPT1 ～ 9 があります(ネット セグメントや信号ポリゴンベタの名前変更など)。

SET CONFIRM OPT2
NAME GND (x y)

最後に、次のコマンドは、

SET CONFIRM OFF

自動確認をオフにします。自動確認が明示的にオフに設定されていない場合は、エディタ ウィンドウがユーザによるインタラクティブな入力を次回受け入れたときに自動的にオフに戻ります。スクリプトの先頭に SET CONFIRM YES を設定してから、複数のコマンドを「サイレント」で実行することはお勧めしません。ベスト プラクティスは、以下の例のように、別の個別のコマンドで明示的に囲むことです。

SET CONFIRM YES
REMOVE _filename_
SET CONFIRM OFF

**このコマンドを使用する場合は、十分に注意してください。メッセージ ダイアログの確認を省略すると、重要なメッセージを見落としてしまう可能性があります。そうなると、想定したとおりの結果が得られないことがあります。**

### EagleRc のパラメータ

機能の細部を調整する必要がある場合があります。たとえば、ユーザによっては、まったく別の動作が必要になる場合があります。これらのパラメータは、ダイアログで使用できるのではなく、eaglerc ファイルのエントリを通じてのみ変更することができます。これを簡単にするために、上記のキーワード内に見つからないパラメータは eaglerc パラメータで参照されるため、SET コマンドを使用して変更することができます。基本的に、eaglerc ファイルのすべてのパラメータはこの方法で変更することができます。パラメータ名は、次に示すように全体を正確に記述する必要があります(大文字と小文字が区別されます)。通常、パラメータ値は「0」または「1」で、それぞれ機能を「オフ」または「オン」にします。画面表示の描画方法に影響を与えるこれらのパラメータを変更すると、ウィンドウの更新が必要になる場合があります。**例**

SET Option.DrawUnprocessedPolygonEdgesContinuous 1;

**Cmd.Copy.ClassicEagleMode**

旧バージョンの EAGLE では、COPY コマンドは図面内のオブジェクトをコピーする目的でのみ使用できました。これは、Windows の動作とは異なります。Windows の場合、COPY は選択したオブジェクトのコピー(つまり、GROUP)をシステムのクリップボードに配置します。バージョン 6 では、EAGLE の COPY コマンドは、現在のグループのコピーをクリップボードに入れることで、他の Windows アプリケーションと同じように動作します。選択したオブジェクトをコピーしたり、ライブラリ間でのライブラリ オブジェクトをコピーする元の機能も、引き続き使用することができます。これは、既存のスクリプトと ULP の動作を維持するために特に重要です。また、Windows ユーザをよくイライラさせることに、EAGLE では CUT コマンドが現在のグループをクリップボードにコピーするだけで、実際には図面からグループを削除しないということがありました。forward-&backannotation を介して接続されているボードと回路図のペアではグループを削除する CUT コマンドがあまり使用されないため、メイン ドロップダウン メニューおよびコマンド ボタン ツールバーから CUT コマンドが削除されました。ただし、コマンド ラインまたはスクリプトでは引き続き完全に使用することができます。このパラメータを「1」に設定すると、COPY コマンドと CUT コマンドの両方の以前の動作が復元されます。この設定は、次回エディタ ウィンドウを開いたときにのみ有効になります。

**Cmd.Delete.WireJointsWithoutCtrl**

[Ctrl]キーを押さなくても、DELETE コマンドでワイヤ接合部を削除できるようにするには、このパラメータを「1」に設定します。

**Cmd.Name.RenameEntireNetByDefault**

ネットが複数のセグメントで構成されている場合、既定では、NAME コマンドは選択されているセグメントにのみ作用します。このパラメータを「1」に設定すると、この動作は既定でネット全体の名前を変更できるようになります。このパラメータは、バスにも適用されます。

**Cmd.Name.RenameEntireSignalByDefault**

信号にポリゴンベタが含まれており、そのポリゴンベタに NAME コマンドが適用された場合、既定ではそのポリゴンベタの名前のみが変更されます。このパラメータを「1」に設定すると、既定で NAME コマンドが信号全体に作用します。

**Cmd.Wire.IgnoreCtrlForRadiusMode**

ワイヤを配置するときに[Ctrl]キーを押して円弧の半径を定義できるという、ワイヤ描画コマンドの特殊なモードを使用しない場合は、このパラメータを「1」に設定します。これにより、ワイヤを描画するすべてのコマンドでこの機能がオフになります。

**ControlPanel.View.AutoOpenProjectFolder**

プログラムの起動時(またはグレーのボタンをクリックしてプロジェクトをアクティブにした場合)にプロジェクト フォルダを自動的に開く機能は、このパラメータを「0」に設定することで無効にできます。

**Erc.AllowUserOverrideConsistencyCheck**

わずかな不整合しかないボードと回路図のペアを処理するために、ERC によってファイルの不整合が検出されていても、強制的にエディタで forward-&backannotation を実行できるダイアログを有効にできます。これは、このパラメータを「1」に設定することで行うことができます。**この設定は、ご自分の責任で行ってください**: 処理中にファイルが破損した場合、その修復にあたって何も行えない可能性があります。結局、ERC によってファイルに不整合があることが**示されて**いるのです。

**Interface.MouseButtonReleaseTimeout**

このコマンドは、インタフェース ボタン(たとえば、ツールバーボタン)をクリックした後に、マウス ボタンをボタンの領域外で放しても、ボタンのアクションをトリガする時間(ミリ秒単位)を設定できます。既定値は 500 です。この機能をオフにするには、この値を 0 に設定します。プログラムの起動時にこのパラメータを 0 に設定した場合、その値を変更しても、変更が有効になるのは、次回プログラムを起動したときです。

**Interface.PreferredUnit**

ダイアログの入力ボックスに数値を表示するときに、最小の小数点以下桁数の表現が選択されるように単位が自動的に決定されます。これをコントロールするには、自動単位決定(既定)の場合はこのパラメータを 0 に、フィート/インチ単位の場合は「1」に、メートル単位の場合は「2」に設定します。

**Interface.UseCtrlForPanning**

画面移動を行うには、マウスの中央ボタン(またはマウス ホイール)を押したままマウスを移動します。旧バージョンでは、代わりに[Ctrl]キーを押してこの操作を行っていました。この古い機能に戻す場合は、このパラメータを「1」に設定します。一部のコマンドで[Ctrl]キーが特殊な機能に使用されるようになりました。そのため、このパラメータを有効にしてこれらの特殊な機能(MOVE コマンドで原点にあるオブジェクトを選択するなど)を使用すると、描画ウィンドウを誤って画面移動してしまうことがあります。

**Option.AutoLoadMatchingDrawingFile**

**Option.DrawUnprocessedPolygonEdgesContinuous**

**Option.RatsnestLimit**

## SHOW

**関数**

 オブジェクトをハイライト表示します。

**構文**

 SHOW ..
 SHOW name..
 SHOW @ name..

**マウス キー**

 *[Ctrl]+左*クリックは、選択したオブジェクトの表示状態を切り替えます。

**関連項目**: [INFO](ECD-CLI-I.html#info)

SHOW コマンドは、オブジェクトをハイライト表示するために使用されます。詳細は、ステータス バーに一覧表示されます。SHOW コマンドで信号とネット全体をハ���ライト表示することができます。バスを選択すると、そのバスに属するすべてのネットもハイライト表示されます。

### クロス プローブ

Forward&Back Annotation がアクティブの場合、SHOW コマンドによってボードでハイライト表示されているオブジェクトは回路図でもハイライト表示され、その逆も同様です。

### さまざまなオブジェクト

SHOW コマンドを使用してさまざまなオブジェクトを選択すると、その 1 つ 1 つのオブジェクトが個別にハイライト表示されます。ハイライト表示するオブジェクトを複数選択するには、[Ctrl]キーを押しながらオブジェクトをクリックします。[Ctrl]キーを押しながら、すでにハイライトされているオブジェクトをクリックすると、そのオブジェクトのハイライトが解除されます。1 行に複数の名前を入力すると、一致するすべてのオブジェクトが同時にハイライト表示されます。

### 小さなオブジェクト

コマンド ラインに @ テキストを指定すると、表示されているオブジェクトの周囲にポインタ長方形が描画されます。これは、ハイライト表示しても目立たない小さなオブジェクトを探す場合に便利です。複数のオブジェクトが表示されている場合、長方形はすべてのオブジェクトの周囲に描画されます。ポインタを表示するために、縮小(または WINDOW FIT コマンドの実行)が必要になる場合があります。リテラル名 @ を含むオブジェクトを表示する場合は、名前を単一引用符で囲む必要があります。

### ワイルドカード

名前パラメータを指定する場合、_ワイルドカード_「*」、「?」、「[]」は、次のような意味を持ちます。

| 

* | 

 | 

任意の数の任意の文字に一致します | 

 |

| 

? | 

 | 

1 文字に一致します | 

 |

| 

[...] | 

 | 

括弧の間の任意の文字に一致します | 

 |

これらの文字のいずれかに正確に一致させる場合、括弧で囲む必要があります。たとえば、abc[*]ghi は abc*ghi に一致しますが、abcdefghi には一致しません。

文字の範囲は[a-z]として指定でき、「a」～「z」の範囲の任意の文字になります。

特殊パターン[number..number]は、[バス名の範囲](ECD-CLI-B.html#x43)を形成するため、回路図ではワイルドカード パターンとして扱われません。

### さまざまなシート上のオブジェクト

名前で指定したオブジェクトが現在の回路図シート上に見つからない場合は、ダイアログが表示され、オブジェクトが見つかったシートのリストが示されます。どのシートにもオブジェクトが見つからない場合、このリストではシート番号が「-」になります。このダイアログは、名前(またはワイルドカード)で指定したオブジェクトのいずれかが現在のシートに見つからない場合にのみ表示されます。指定したすべてのオブジェクトが現在のシートで見つかった場合、(他のシートにもオブジェクトが存在している場合でも)ダイアログは表示されません。ダイアログが表示されると、そのダイアログには現在のシート上のものも含め見つかったすべてのオブジェクトが含まれています。

### 例

SHOW IC1

IC1 は、ハイライト表示され、SHOW コマンドが終了するか別の名前を入力するまでハイライト表示されたままになります。

SHOW IC*

名前が「IC」で始まるオブジェクトをすべてハイライト表示します。

## SIGNAL

**関数**

 信号を定義します。

**構文**

 SIGNAL ..
 SIGNAL signal_name ..
 SIGNAL signal_name element_name pad_name..;

**関連項目**: [AUTO](ECD-CLI-A.html#auto)、[ROUTE](ECD-CLI-R.html#route)、[NAME](ECD-CLI-N.html#name)、[CLASS](ECD-CLI-C.html#class)、[LINE](ECD-CLI-L.html#line)、[RATSNEST](ECD-CLI-R.html#ratsnest)、[EXPORT](ECD-CLI-E.html#export)

SIGNAL コマンドは、信号(さまざまなフットプリント間の接続)を定義するために使用されます。ユーザは、少なくとも element_ name と pad_ name という 2 つのペアを定義する必要があります。そうしないと、エアワイヤを生成できません。

### マウス入力

これを行うには、接続する要素のパッド(または SMD)を(マウスで)段階的に選択します。EAGLE は、未配線画層において、パーツ信号をエアワイヤとして表示します。signal_name を入力すると、指定した名前が信号に割り当てられます。

### テキスト入力

(キーボードまたはスクリプト ファイルを使用して)テキスト入力で信号を完全に定義することもできます。コマンド

SIGNAL GND IC1 7 IC2 7 IC3 7;

IC1 ～ 3 のパッド 7 を接続します。ネットリスト全体を入力するために、*.scr という拡張子でスクリプト ファイルを生成することもできます。このファイルには、必要なすべての SIGNAL コマンドを上記の形式で含める必要があります。

### 接続チェック

既にさまざまな信号に属しているパッド(または SMD)を接続するために SIGNAL コマンドを使用している場合は、ポップアップ メニューが表示され、それらの信号をまとめて接続するかどうか、および信号にどの名前を付けるかを尋ねられます。

### アウトライン データ

特別な信号名 *OUTLINES* は、アウトライン データの生成に使用される特定のプロパティを信号に付与します。その用途以外では、この名前を使用しないでください。

## SIM

**関数**

 SPICE シミュレーションを実行します。

**構文**

 SIM  

**関連項目**: [EXPORT](ECD-CLI-E.html#export)、[INFO](ECD-CLI-I.html#info)、[ADDMODEL](ECD-CLI-A.html#addmodel)、[MAPTOMODEL](ECD-CLI-M.html#maptomodel)、[SOURCESETUP](#sourcesetup)、[VALUE](ECD-CLI-V.html#value)。 

### SPICE シミュレーション

#### 基本操作

SIM コマンドは、EAGLE から SPICE シミュレーションを開始するために使用します。パーツはシミュレーション対応にする必要があり、Eagle には **ngspice-simulation** という管理ライブラリが用意されています。SIM コマンドを開始するには、EAGLE コマンド ラインに「SIM」と入力するか、[SIM]コマンドをクリックします。設定、ネットリスト、シミュレーション結果、およびプロットが、シミュレーション ダイアログの対応するタブに表示されます。  

サポートされるシミュレータは次のとおりです。  

- Ngspice (バージョン 26)、[http://ngspice.sourceforge.net/](http://ngspice.sourceforge.net/)

#### パッシブ サイン規則

EAGLE では、パッシブ サイン規則(PSC)を使用して、電流がデバイスの正端子に流れ込むときには電力が正の値であることを示します。つまり、通常、荷重の電力散逸は正となり(電力を消費し)、ソースの電力散逸は負となります(電力を供給します)。  

#### シミュレーション ダイアログのタブ

SIM コマンドが起動すると、シミュレーション ダイアログがポップアップします。ダイアログには、次のタブがあります。

- **[設定]:** 特定のタイプのシミュレーションを設定、保存、および実行します。

- **[ネットリスト]:** 回路ネットリストを表示および編集し、手動で編集したネットリストからシミュレーションを実行します。

- **[シミュレータ出力]:** 未加工のシミュレータ出力を表示します。

- **[印刷]:** 信号プロッタでデータを表示および操作します。

#### シミュレーション対応のライブラリ パーツ

シミュレータを使用するには、すべてのパーツがシミュレーションに対応している必要があります。つまり、パーツが EAGLE シミュレーション環境で機能するために必要な特定の属性があります。**ngspice-simulation** というライブラリは、EAGLE に用意されているシミュレーション対応の基本的なライブラリです。このライブラリのすべてのパーツをシミュレーションで使用できます。 

**接地**
有効な接地シンボルには SPICEGROUND という属性があり、空の値になっています。シミュレータが誤りなく計算するには、接地シンボルが少なくとも 1 つ存在し、回路内の特定のネットに接続されている必要があります。  

**初期条件**
シミュレーション対応パーツの値と属性を設定できるだけでなく、パーツの初期条件(コンデンサの初期電圧など)も設定できます。初期条件は、属性として追加されます(次を参照)。  

**SPICE マッピング**
パーツが SPICE シミュレーションに対応している場合、各ピンには SPICE モデルにマッピングされる順序が設定されている必要があります。このプロパティは、ピンを SPICE サブ回路、モデル カード、または固有のモデルの正しい入力にマッピングします。たとえば、5 つのピンがあるオペアンプ パーツを考えてみます。オペアンプのライブラリ シンボルでは、5 つのピンが必要です。これに対応して、SPICE モデルでは 5 つの入力が必要です。各ピンの順序値は、ピンをモデル入力パラメータにマッピングする、1 から始まる数値です。このマッピングを行うには、ADDMODEL を使用してパーツを SPICE 互換にし、ライブラリ デバイス エディタおよび回路図で MAPTOMODEL コマンドを使用します。  

ライブラリにマッピングされたパーツは、ライブラリ パーツに SPICE モデルを保存します。パーツにライブラリからのマッピングが事前に設定されていたかどうかにかかわらず、ユーザは常に回路図内でパーツを再マッピングすることができます。この場合、パーツ自体はその情報を保存し、ライブラリ情報をオーバーライドします。パーツ ピンをモデル ピンにマッピングできるように、[ADDMODEL](ECD-CLI-A.html#addmodel) コマンドおよび [MAPTOMODEL](ECD-CLI-M.html#maptomodel) コマンドにはマッピング インタフェースが用意されています。  

**パーツ属性**: ユーザが回路図でパーツをマッピングする場合、次の属性が使用されます。

- **SPICEMODEL:** 空であるか、.mdl の拡張子がない有効な SPICE モデル ファイルの名前。

- **SPICEPREFIX:** このタイプのデバイスに使用される 1 文字の有効な SPICE 接頭表記。(例: R、L、C、X など)

- **SPICEINITCOND:** SPICE パーツの初期条件(コンデンサの場合は電圧、インダクタの場合は電流)

- **SPICEEXTRA:** これらは、ネットリストでデバイス モデル名の後に続けるオプションの値です。通常、物理パラメータ指定子(L=10 W=20 など)に使用されます。

- ***EXTERNAL***: シミュレーション互換であるもののパッケージがないパーツのオプションの属性で、値を空にする必要があります。

#### SPICE モデルの使用方法

一部のコンポーネント(トランジスタ、ダイオード、サブ回路など)は、定義上、SPICE モデルと SPICE モデル カードを使用する必要があります。コンポーネントにモデルを使用する場合、次の 2 つの**モデルを使用する**ステップがあります。

- MAPTOMODEL コマンドを使用して、パーツに適切な属性を追加します。

- モデル ファイルをグローバルな **/models/** フォルダまたは回路図と同じフォルダに配置します。

検索順序は次のとおりです。

- パーツがライブラリにマッピングされていた場合、パーツはライブラリ内に存在する必要があります。存在しない場合は ...

- モデル ファイルがまず、回路図デザインが存在するフォルダ内で検索されます。

- そのフォルダに存在しない場合は、ユーザが設定可能な /models/ フォルダが検索されます。

上記 3 つの場所のいずれかでモデルが見つからない場合、モデルがネットリストに追加されないため、シミュレーションは失敗します。シミュレーション結果には、モデルが見つからなかったことが示されます。

#### ネットリスト

EAGLE では、[EXPORT SPICENETLIST](ECD-CLI-E.html#x63)コマンドを呼び出すことで、SPICE 互換のネットリストをいつでも生成することができます。ネットリストは、回路図と同じフォルダに保存され、**.sch.net** という拡張子が付けられます。 

拡張子が ***.sch.final.net** のファイルは、同じネットリストであるものの、最新のシミュレーション設定が含まれています。これらはシミュレータに送信されるファイルで、***.sim** ファイルはシミュレータの未加工の出力です。  

*.**sim.proc.log** ファイルは、シミュレータを実行したときのシステムからのレスポンスで、システムまたはセットアップに問題がある場合に役立ちます。  

**ネットリストの機能**
ネットリストは、Eagle によって生成され、シミュレーションに使用されます。ただし、手動でネットリストを変更し、手動で編集したネットリストに基づいてシミュレーションを実行できます。ネットリストを編集するには、ネットリスト タブで変更を加えます。*ネットリスト タブから*[シミュレート]をクリックすると、手動で編集したネットリストを使用してシミュレートされます。

#### 印刷

シミュレーションの結果を印刷できる場合(AC、DC、TRAN)、印刷タブが自動的に開き、データが印刷されます。デザインにプローブが含まれていない場合、名前の付いたすべてのノードのノード電圧と、すべてのソースの電流が表示されます。デザインにプローブがある場合、プローブされた信号のみが表示されます。 

印刷上の特定の領域を拡大するには、マウスでウィンドウ ズームをクリック アンド ドラッグします。縮小するには、印刷上でダブルクリックします。マウスを動かすとすべての信号がトレースされ、対応する値が印刷の凡例に表示されます。

#### ソースの設定

接頭表記が V または I (SPICE に依存しないソース)である SPICE シミュレーション対応のパーツに対して、SOURCESETUP コマンドを使用すると、ソース パーツの DC、AC、および非定常関数パラメータを設定するためのユーザ インタフェースが提供されます。現在、正弦関数、指数関数、およびパルス関数がサポートされています。

#### サポートされている機能

- ネットリスト内での制御文とスクリプトの使用はサポートされていません。

- シミュレーション タイプ: AC、DC、TRAN、OP

- 有効な SPICE サブ回路モデルとモデル カードの使用。サブ回路には、モデルやその他のサブ回路を含めることができます。

- シミュレーションの前にネットリストの検証が行われ、無効なネットリスト行があればユーザが参照できるように記録されます。

- シミュレーション結果は、未加工のテキスト形式で印刷および提供されます。

- ライブラリ内でのパーツのマッピングがサポートされ、SPICE モデルは容易に共有できるようにライブラリ パーツに保存されます。

- 回路図内でのパーツのマッピングがサポートされ、パーツのマッピングはライブラリでのマッピングよりも優先されます。

- 事前に設定された SPICE 互換のパーツが、ngspice-simulation ライブラリに用意されています。

- シミュレーション設定は、回路図と同じフォルダ内の .simcfg ファイルに対して自動的にロードおよび保存されます。

- SPICE モデル ファイルがサポートされ、ユーザ ファイルを適切にロードするには、それが使用されている回路図と同じフォルダか、/models/ アプリケーション フォルダにユーザ ファイルを保存する必要があります。

## SLICE

**関数**

 行を分割します。

**構文**

 SLICE [width] [ripup]    

**関連項目**: [DELETE](ECD-CLI-D.html#delete)、[RIPUP](ECD-CLI-R.html#ripup)

SLICE コマンドは、線分を 2 つに切断するために使用されます。これを使用すると、再配線のためにボード トレースまたは回路図ネットを分離することができます。幅は、スライスで線分をどの程度除去するかを定義します。除去される線分の量は、幅設定または線幅の 2 倍より大きい値になります。表示されているレイヤの線分のみをスライスすることができます。エア ラインとポリゴンベタはスライスされません。

スライスには、3 つのリップアップ オプションがあります。1)リップアップなしのスライス(既定) 2)スライスの左のリップアップ セグメント 3)スライスの右のリップアップ セグメント。これらのオプションは、パラメータ バーのアイコンとして使用することができます。また、コマンド ラインから SLICE コマンドを呼び出して、オプションでリップアップをコントロールすることができます。リップアップなしの場合は「SLICE None」、スライス線の左のリップアップの場合は「SLICE L」、スライス線の右のリップアップの場合は「SLICE R」です。リップアップ方向は、スライス線の方向を基準にしています。スライス線は、始点から終点までの矢印またはベクトルに似ています。

## SMD

**関数**

 フットプリントに SMD パッドを追加します。

**構文**

 SMD [x_width y_width] [-roundness] [orientation] [flags] ['name'] ..

**マウス キー**

 *中央*クリックは、レイヤを選択します。
 *右*クリックは SMD を回転します。
 *[Shift]+右*クリックは、回転方向を反転します。

**関連項目**: [PAD](ECD-CLI-P.html#pad)、[CHANGE](ECD-CLI-C.html#change)、[NAME](ECD-CLI-N.html#name)、[ROUTE](ECD-CLI-R.html#route)

SMD コマンドは、表面実装型デバイスのパッドをフットプリントに追加するために使用されます。SMD コマンドがアクティブな場合、SMD 記号がカーソルにアタッチされます。マウスの左ボタンを押すと、現在の位置に SMD パッドが配置されます。数値を入力すると、SMD パッドの X と Y の幅が変更されます。最大幅は、200 mm (7.7840 インチ)です。これらのパラメータは、後続の SMD コマンドでも既定値のままとなり、変更するには CHANGE コマンドを使用します。マウスの中央ボタンを押すと、SMD パッドが描画されるレイヤが変更されます。

orientation ([ADD](ECD-CLI-A.html#add) の説明を参照)は、範囲 R0...R359.9 の任意の角度です。S フラグと M フラグはここでは使用できません。

### 丸み

丸みは、0 から 100 までの整数として入力する必要があります。幅パラメータと区別するには、負の符号を付けます。値を 0 にすると完全に長方形の SMD になり、値を 100 にすると SMD のコーナーが完全に丸くなります。コマンド

SMD 50 50 -100 '1' 

であれば、特定のマウス クリック位置に「1」という名前で完全に丸い SMD を作成します。これを使用して、BGA (ボール グリッド アレイ)パッドを作成することができます。

### 任意のパッド形状

標準の SMD パッド形状が特定のフットプリントに十分でない場合は、SMD パッドの周囲にポリゴンベタを描画するか、一端を SMD パッドに接続したワイヤを描画することで、任意の SMD パッド形状を作成することができます。次の条件が適用されます。

- パッドの中心がポリゴンベタ ワイヤの中心線で定義された領域内にある場合、ポリゴンベタは同じ画層上の SMD パッドに接続されているとみなされます。

- ワイヤの終点の 1 つがパッドの中心と一致する場合、ワイヤは同じ画層上の SMD パッドに接続されているとみなされます。このようなワイヤのもう一方の端点に接続されているワイヤも、電気的にパッドに接続されています。

- パッドごとに **1** つのポリゴンベタのみが考慮されます。複数のポリゴンベタが同じパッドに接続されている場合は、DRC エラーが発生します。

- パッドに接続されているポリゴンベタは、その信号を配線するときにオートルータによって無視されます。これらのポリゴンベタは、他の信号を配線するときに障害物とみなされます。

- パッドに接続されているワイヤは、オートルータによって他の信号ワイヤと同様に処理されます。ただし、ワイヤを分割できない場合は例外です。

- はんだストップ マスクとクリーム マスクは、パッド自体に対してのみ生成されます。はんだストップ マスクまたはクリーム マスクが追加で必要な場合は、それぞれの画層に明示的に描画する必要があります。

- 熱解析を生成する際に、追加のポリゴンベタ形状が考慮されます。

- ポリゴンベタまたはワイヤがフットプリント内の複数のパッドに接続されている場合、それらのパッドのうち 1 つのみがポリゴンベタまたはワイヤに電気的に接続されているとみなされます。その他のパッドは、すべてデバイス内の同じピンに接続されていない限り、DRC エラーを引き起こします。

- C 形状のポリゴンベタがパッドに接続されているために、C 領域内でボードの信号ポリゴンベタに「孤立」が存在する場合、その信号ポリゴンベタの Orphans パラメータがオンかオフかに関係なく、そのような孤立は表示されなくなります。

### 名前

SMD 名は自動的に生成されますが、NAME コマンドで変更することもできます。SMD コマンドに名前を含めるには、単一引用符で囲みます。

### フラグ

次の_フラグ_を使用すると、SMD の外観をコントロールできます。

| 

NOSTOP | 

 | 

はんだストップ マスクを生成しない | 

 |

| 

NOTHERMALS | 

 | 

熱解析を生成しない | 

 |

| 

NOCREAM | 

 | 

クリーム マスクを生成しません。 | 

 |

既定では、SMD が必要に応じてはんだストップ マスク、クリーム マスク、およびサーマルを自動的に生成します。ただし、SMD によってはこれを行わない方が望ましい特殊なケースもあります。上記の NO... フラグは、これらの機能を抑制するために使用できます。
新たに SMD コマンドを開始すると、すべてのフラグが既定値にリセットされます。コマンド ラインにフラグを指定すると、この SMD コマンド内に配置した次のすべての SMD にそのフラグが適用されます。

## SMDARRAY

**関数**

 SMD の配列を作成します。

**構文**

## SOURCESETUP

**機能**

 シミュレーション互換のソース パーツに DC、AC、および一時パラメータを設定するために使用します。

**構文**

 SOURCESETUP name

**関連項目**: [ADDMODEL](ECD-CLI-A.html#addmodel)、[SIM](#sim)、[DSOURCESETUP](ECD-CLI-D.html#dsourcesetup)

このコマンドは、SPICE 互換のソース パーツの値を設定するために使用します(SPICE 接頭表記が V または I のパーツに有効)。DC、AC、非定常関数(正弦、指数、およびパルス)の値は、UI 内から使用することができます。ユーザは、常に他の有効なソース値に対して値を直接設定することができます。コマンドは、回路図内のパーツ インスタンスのコンテキスト メニュー項目、コマンド ライン、およびツールバーのコマンド ボタンを介して使用できます。パーツがまだ SPICE 互換でない場合、ADDMODEL はパーツを設定する前にパーツの変換を開始します。

## SPLIT

**関数**

 ワイヤとポリゴンベタ エッジをセグメントに分割します。

**構文**

 SPLIT [curve | @radius] ..

**マウス キー** 

 *右クリック*すると、ワイヤ曲げスタイルが変更されます(「[SET Wire_Bend](ECD-CLI-S.html#x129)」を参照)。
 *[Shift]+右*クリックは、曲げスタイルの切り替え方向を反転します。
 *[Ctrl]+右*クリックは、対応する曲げスタイルを切り替えます。
 *[Ctrl]+左*クリックは、配線の終点の配置時に円弧の半径を定義します。

**関連項目**: [MITER](ECD-CLI-M.html#miter)、[MOVE](ECD-CLI-M.html#move)、[OPTIMIZE](ECD-CLI-O.html#optimize)、[SET](#set)

SPLIT コマンドは、曲げを導入する場合などに、ワイヤ(またはセグメント)またはポリゴン エッジを 2 つのセグメントに分割するために使用されます。つまり、SPLIT コマンドの実行中に、マウスを使用して移動できるパーツにワイヤを分割することができます。クリックすると、ワイヤが分割される点が定義されます。2 つの新しいセグメントのうち短い方は、現在のワイヤ曲げルールに従うため、さらに 2 つのセグメントになる場合があります(「SET Wire_Bend」を参照)。長い方のセグメントは、次の終点に向かう直線セグメントになります。

*curve* または *@radius* パラメータを指定すると、円弧をワイヤ セグメントの一部として描画できます([LINE](ECD-CLI-L.html#line) コマンドの詳細な説明を参照)。

SPLIT コマンドを実行すると、位置合わせされたセグメントは自動的に再結合されます。以下の場合、これが無効になります。コマンド

SET OPTIMIZING OFF;

が以前に指定されている場合、またはワイヤが同じ場所で 2 回クリックされている場合。この場合、分割点は残ったままになるため、たとえば、**セグメントの幅を小さくする**ために使用できます。このためには、SPLIT コマンドを選択し、マウスを 2 回クリックして小さくするワイヤの部分をマークし、次のコマンドを入力します。

CHANGE WIDTH width

その後、セグメントをクリックすると、変更が完了します。
