## UNDO

**関数**

 前のコマンドをキャンセルします。

**構文**

 UNDO
  UNDO LIST

> **Note:** **関連項目**: [REDO](ECD-CLI-R.html#redo)、[SET](ECD-CLI-S.html#set)、Forward&Back Annotation

UNDO コマンドを使用すると、前に実行したコマンドをキャンセルできます。これは、誤って削除を実行してしまった場合に特に便利です。UNDO コマンドを複数回実行すると、直前の EDIT、OPEN、REMOVE コマンドに達するまで、実行したコマンドの数だけキャンセルされます。

UNDO コマンド バッファを実行すると、メモリの消費量が増えます。メモリが不足している場合は、SET コマンドを使用してこの機能をオフに切り替えることができます。

SET UNDO_LOG OFF;

 UNDO/REDO は Forward&Back Annotation 内に完全に統合されています。

### 元に戻すバッファのダイアログ

UNDO コマンドに LIST オプションを指定すると、元に戻すバッファの内容全体を一覧表示するダイアログが開きます。元に戻す/やり直しステップのリストをナビゲートするには、リストの区切り記号をクリック アンド ドラッグするか、前後にナビゲートする任意のステップを直接クリックします。現在の区切り位置とクリックしたリスト項目の間に複数のステップがある場合は、その間のすべてのステップが適切な順序で実行されます。リスト内で上に移動すると、UNDO が実行され、下に移動すると REDO が実行されます。  

各リスト項目のアイコンは、この特定のコマンドが実行された図面を示します。このダイアログを「OK」で確定すると、図面はリストで選択した状態のままになります。このダイアログをキャンセルすると、ダイアログを開く前の状態に戻ります。

> **Note:** **注意:** これは非常に強力なツールです。UNDO リストの内容全体を元に戻し(1 回のマウス クリックで可能)、任意の新規コマンドを実行すると、その時点で元に戻すバッファが切り捨てられるので、元に戻す方法はありません。そのため、これは慎重に使用してください。

## UPDATE

**関数**

 ライブラリ オブジェクトを更新します。

**構文**

 UPDATE
  UPDATE;
  UPDATE library_name..;
  UPDATE footprint_name@library_name..;
  UPDATE +@ | -@ [library_name..];
  UPDATE old_library_name = new_library_name;

> **Note:** **関連項目**: [ADD](ECD-CLI-A.html#add)、[REPLACE](ECD-CLI-R.html#replace)

UPDATE コマンドは、基板または回路図内のパーツをそれぞれのライブラリ オブジェクトと比較してチェックし、両者が異なる場合には自動的に更新が行われます。ライブラリ エディタから UPDATE を呼び出すと、ロードされたライブラリ内のフットプリントが指定したライブラリから更新されます。

パラメータを指定せずに UPDATE コマンドをアクティブにすると、更新するライブラリを選択するためのダイアログが表示されます。

ライブラリを 1 つまたは複数指定した場合は、それらのライブラリのパーツのみがチェックされます。ライブラリ名は、プレーンなライブラリ名(「ttl」や「ttl.lbr」など)、完全表記のファイル名(「/home/mydir/myproject/ttl.lbr」や「../lbr/ttl」など)、またはダウンロードした管理ライブラリの URN (「urn:adsk.eagle:library:42」など)のいずれかにすることができます。

library_name に空白が含まれている場合は、単一引用符で囲む必要があります。

### ボードまたは回路図での更新

コマンドにパラメータが指定されていない場合、「;」で終了すると、すべてのパーツがチェックされます。最初のパラメータが「+@」の場合、指定したローカル ライブラリ(または何も指定されていない場合はすべてのローカル ライブラリ)の名前に「@」文字と番号が付加されます。これを使用すると、図面に含まれているライブラリと同じ名前の新しいライブラリからパーツが図面に追加されたときに、既存のライブラリが変更されないようにすることができます。「@」文字とその後に続く数字で終わる既存のライブラリ名は変更されず、管理ライブラリの名前も変更されません。

最初のパラメータを「-@」にした場合は、その指定したライブラリ(または、何も指定していない場合はすべてのライブラリ)の「@」文字(後に数字が続く)がライブラリ名から削除されます。これは、当然ながら、その新しい名前のローカル ライブラリが図面にまだ存在していない場合にのみ機能します。

「UPDATE +@;」の後に「UPDATE -@;」を続けた場合(およびその逆の場合)、必ずしも元のライブラリ名のセットになるわけではないことに注意してください。名前を処理する順序が、ライブラリを図面ファイルに保存した順序によって異なるためです。

ボードまたは回路図の図面に保存されているライブラリはベース名(「ttl」など)で識別され、管理ライブラリは URN (「urn:adsk.eagle:library:462」など)で識別されます。更新元のライブラリと更新先のライブラリが一致する場合、その 2 つの URN が一致するケース(管理ライブラリ)、または名前が一致して URN がないケース(ローカル ライブラリ)に最初に優先権が与えられます。このような一致がない場合、UPDATE は名前が一致し、2 つのライブラリのいずれかに URN がないケースに戻ります。(URN が異なる管理ライブラリ間では、名前が同じでも一致は行われません。)一致の可能性が複数ある場合は、使用するライブラリを選択するためのダイアログが表示されます。図面に保存されているライブラリ名は、大文字と小文字が区別され、ローカル ライブラリのパスが考慮されないことに注意してください(コマンド ラインでパスを指定した場合を除く)。特定のライブラリが現在「使用中」であるかどうかは関係ありません。ただし、管理ライブラリが使用されるのは、そのライブラリをダウンロードしていた場合のみです。ライブラリが見つからない場合、そのライブラリの更新は実行されません。ライブラリをコマンド ラインに入力した場合は、エラーが表示されるだけです。ボードまたは回路図に存在するすべてのライブラリを更新する場合は、ライブラリが見つからなくてもエラーが表示されません。

アクティブな Forward&Back Annotation を介して接続されている回路図またはボードで UPDATE コマンドを使用すると、回路図とボードの両方に対して機能します。

ゲート、ピン、またはパッドを名前または座標でマッピングするかどうかを指定することが必要になる場合があります。これが必要になるのは、対応するライブラリ オブジェクトが名前変更されるか移動された場合です。変更が多すぎる場合(ピンの名前を変更し、かつそのピンを移動する場合など)には、自動更新ができない可能性があります。この場合は、2 つのステップ(1 つは名前の変更、もう 1 つは移動)でライブラリの変更を行うか、ライブラリ オブジェクト全体に別の名前を付けることができます。

old_library_name = new_library_name (「=」文字の前後に少なくとも 1 つの空白が必要)を指定すると、UPDATE コマンドは現在のボードまたは回路図で *old_library_name* という名前のライブラリを特定して、*new_library_name* の内容で更新します。*old_library_name* はパスを含まない純粋なライブラリ名である必要があり、*new_library_name* は絶対パス名または URN です。更新が正常に実行された場合、それに応じて現在のボードと回路図ファイルのライブラリの名前も変更されます。したがって、この操作全体は、当然ながら、現在のボードまたは回路図で *new_library_name* がまだ使用されていない場合にのみ可能です。

> **Note:** **注: ボードまたは回路図でライブラリの更新を実行したら、[デザイン ルール チェック](ECD-CLI-D.html#x58)(DRC)と[電気的ルール チェック](ECD-CLI-E.html#x61)(ERC)を必ず実行してください。**

### ライブラリでの更新

ライブラリを更新すると、そのライブラリ内のすべてのフットプリントが、指定されたライブラリのバージョンに置き換えられます。フットプリント名(footprint_name@library_name)を指定すると、特定のフットプリントのみを置換できます。

## USE

**関数**

 ライブラリに使用のマークを付けます。

**構文**

 USE
  USE -*;
  USE library_name..;
  USE -library_name..;

> **Note:** **関連項目**: [ADD](ECD-CLI-A.html#add)、[REPLACE](ECD-CLI-R.html#replace)

USE コマンドは、後で [ADD](ECD-CLI-A.html#add) コマンドまたは [REPLACE](ECD-CLI-R.html#replace) コマンドで使用するライブラリをマークします。

パラメータを指定せずに USE コマンドをアクティブにすると、ライブラリ管理を許可するダイアログが表示されます。ここでは、使用するライブラリをマークし、ライブラリを使用対象から外し、管理ライブラリをダウンロードして更新することができます。

特別なパラメータ -* を指定すると、以前にマークされたすべてのライブラリがドロップされます。-library_name を使用すると、library_name というライブラリがドロップされます。

library_name には、ライブラリの完全表記の名前を使用することも、ワイルドカードを含めることもできます。また、ダウンロードした管理ライブラリの URN にすることもできます。library_name がフォルダの名前である場合は、そのフォルダのすべてのライブラリがマークされます。

接尾表記 .lbr は省略することができます。

デバイスまたはパッケージを図面に追加すると、そのオブジェクトのライブラリ情報全体が図面ファイルにコピーされるため、後で図面を変更する場合にライブラリが必要ないことに注意してください。

ライブラリを変更しても、既存の図面には影響しません。変更したライブラリのパーツを更新する場合は、[UPDATE](#update) コマンドを参照してください。

### コントロール パネルを介したライブラリの使用

コントロール パネルで容易に使用できるようにライブラリをマークするには、アクティブ化アイコンをクリックするか(色が変わって、このライブラリが使用中であることが示されます)、ライブラリのコンテキスト メニューから[使用]を選択します。コントロール パネルの[ライブラリ]エントリのコンテキスト メニューを使用すると、_すべて_のライブラリを使用することも、ライブラリの_いずれも_使用しないようにすることもできます。

### 使用しているライブラリおよびプロジェクト

現在使用しているライブラリは、プロジェクト ファイルに保存されます(プロジェクトが現在開いている場合)。

### 例

| 

USE | 

 | 

ライブラリを選択するためのファイル ダイアログを開きます。 | 

 |

| 

USE -*; | 

 |

 |

| 

USE demo trans*; | 

 | 

ライブラリ demo.lbr と名前が trans*.lbr に一致するすべてのライブラリをマークします。 | 

 |

|

 | 

URN が urn:adsk.eagle:library:462 の管理ライブラリをマークします。 | 

 |

|

 |

 |
