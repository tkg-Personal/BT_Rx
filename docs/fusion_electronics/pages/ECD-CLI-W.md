## WINDOW

**関数**

 図面を拡大縮小します。

**構文**

 WINDOW;
  WINDOW ;
  WINDOW ;
  WINDOW
  WINDOW scale_factor
  WINDOW FIT
  WINDOW LAST
  WINDOW FLIP

**マウス キー**

 *左クリック & ドラッグ*は、長方形のウィンドウを定義します(「 ;」のショートカット)。

**キーボード**

 Alt+F2: WINDOW FIT 図面を画面にフィットさせます。
  F2: WINDOW; 画面を再描画します。
  F3: WINDOW 2 2 倍拡大します。
  F4: WINDOW 0.5 2 倍縮小します。
  F5: WINDOW (@); カーソル位置が新しい中心になります(コマンドがアクティブな場合)。

WINDOW コマンドは、図面を拡大縮小したり、画面上の図面の位置を変更するために使用します。このコマンドは、最大 3 回のマウス クリックで使用することができます。それより少なくする場合は、セミコロンで終了する必要があります。

### 画面の更新

WINDOW コマンドの後にセミコロンを続けると、画面の中心や尺度を変更することなく画面が再描画されます。これは、エラー メッセージが図面の一部をおおっている場合に便利です。

### 新しい中心

点を 1 つ指定して WINDOW コマンドを実行すると、その点が図面の新しい画面表示の中心になります。図面の尺度は変わりません。また、作業領域のスライダを使用して、図面の表示領域を移動することもできます。ファンクション キー[F5]を押すと、カーソルの現在の位置が新しい中心になります。

### コーナー点

2 つの点を指定して WINDOW コマンドを実行すると、指定した点を対角コーナーとした長方形が定義されます。この長方形が画面全体に拡大され、図面の指定した部分が拡大表示されます。

### 新しい中心と拡大縮小

3 つの点を指定して WINDOW コマンドを使用することができます。最初の点は図面の新しい中心を定義し、表示は他の点との間隔の比率に応じて大きくなるか小さくなります。拡大するには、点 1 と点 3 の間の距離を点 1 と点 2 の間の距離よりも大きくする必要があります。縮小するには、点 1 と点 2 の間に点 3 を配置します。

### ズームインおよびズームアウト

WINDOW 2;

要素を 2 倍の大きさにします。

WINDOW 0.5;

要素のサイズを 2 倍小さくします。WINDOW コマンドの引数として整数または実数を指定すると、入力した分だけ図面の表示を拡大または縮小できます。ウィンドウの中心は同じままになります。

### 図面全体

WINDOW FIT;

画面に図面全体をフィットさせます。

### 前のウィンドウに戻る

WINDOW LAST;

直前のウィンドウ選択に戻ります。拡大縮小専用の WINDOW コマンドおよびマウスによるウィンドウ選択の変更を除き、ウィンドウ選択はすべての WINDOW コマンドで保存されます。

### ウィンドウの反転

WINDOW FLIP;

ボードの下側のパースからボードを表示および編集することができます。

### 非常に大きな倍率

既定では、直径が 1 mm (約 40 ミル)の領域がエディタ ウィンドウ全体に表示されるように、最大倍率が制限されています。それ以上に拡大する必要がある場合は、[オプション]/[ユーザ インタフェース]/[倍率を制限]をオフにして、最も細かいエディタ グリッドが表示されるまで拡大することができます。図面を非常に大きく拡大すると、次のことが起きることがあります。

- ベクトル フォントを使用していないテキストは、エディタ ウィンドウよりも大きい場合には表示されないことがあります。

- 円と円弧が接近するため、どちらも正確な位置に表示されない場合があります(特に、幅が非常に小さい場合)。

- 全体を拡大するときに最も細かいグリッドが表示されるかどうかは、画面解像度、エディタ ウィンドウのサイズ、および[オプション]/[設定]/[その他]/[最小表示グリッド サイズ]の値によります。

### パラメータのエイリアス

パラメータのエイリアスを使用すると、WINDOW コマンドに特定のパラメータ設定を定義することができます。このパラメータ設定は、後で名前を指定して参照することができます。また、エイリアスには、[ウィンドウ選択]ボタンをクリックし、リストがポップアップされるまでマウス ボタンを押したままにしてアクセスすることもできます。ボタンを右クリックしても、このリストがポップアップ表示されます。これらのエイリアスを処理するための構文は次のとおりです。

WINDOW = *name* *parameters*

指定された *name* でエイリアスを定義し、指定された *parameters* に展開されるようにします。*name* は任意の数の文字、数字、および下線で構成され、大文字と小文字は区別されません。文字または下線で始まる必要があり、いずれかのオプション キーワードであってはなりません。

WINDOW = *name* @

指定した *name* のエイリアスを現在のウィンドウ選択に拡大するように定義します。

WINDOW = ?

現在のウィンドウ設定のエイリアスを定義するための名前を入力するように求めます。

WINDOW = *name*

指定した *name* でエイリアスとして定義されるウィンドウを選択できます。

WINDOW = *name*;

指定された *name* を持つエイリアスを削除します。

WINDOW *name*

指定した *name* のエイリアスを拡大し、その結果もたらされるパラメータのセットを指定して WINDOW コマンドを実行します。*name* は省略可能で、エイリアスの前後に他のパラメータがあってもかまいません(他のエイリアスも同様)。*name* が省略形である場合、コマンドの他のパラメータ名よりもエイリアスが優先されることに注意してください。

例: WINDOW = MyWindow (0 0) (4 3);

エイリアス「MyWindow」を定義します。これは、

WINDOW myw などで使用すると、指定したウィンドウ領域まで拡大します。エイリアスの省略形の使用と、大文字と小文字が区別されていないことに注意してください。

## WRITE

**関数**

 現在の図面、ライブラリまたはデザイン ブロックを保存します。

**構文**

 WRITE;
  WRITE name
  WRITE @name
  WRITE 7 [ name ]
  WRITE [ DBL ] name

WRITE コマンドは、図面、ライブラリ、またはデザイン ブロックを保存するために使用します。「name」を入力すると、その新しい名前でファイルが保存されます。別のフォルダに保存する場合は、パス名を含めたファイル名を入力することもできます。パス名を指定しない場合、ファイルはプロジェクト フォルダまたはデザイン ブロック フォルダ(オプション DBL)に保存されます。

新しい名前の前に @ を付けると、ロードした図面の名前もそれに応じて変更されます。対応するボード/回路図もこの名前で自動的に保存され、UNDO バッファがクリアされます。

メニューから WRITE を選択すると、ポップアップ ウィンドウが表示され、使用する名前を尋ねられます(既定は現在の図面名)。[OK]ボタンをクリックして、この名前を編集および受け入れることができます。[Esc]キーを押すか[キャンセル]ボタンをクリックすると、WRITE コマンドがキャンセルされます。

ボード図面と回路図図面間で Forward&Back Annotation の一貫性を保つために、WRITE コマンドは次の機能も備えています。

- ボード/回路図に変更を加えた場合に、同じ名前で保存すると、対応する回路図/ボードも保存されます。

- ボード/回路図を別の名前で保存すると、その別の名前で回路図/ボードも保存するかどうかを尋ねられます。

- 別の名前で図面を保存しても、「変更済み」フラグはクリアされません。

### EAGLE 7.x 用の保存

WRITE の後に 7 を入力すると、ファイルは EAGLE 7.x 用に保存されます。EAGLE 8.x で追加されたすべての XML 属性は省略され、EAGLE 7.x でファイルを開けるようにその他の調整が行われます。管理ライブラリの URN やその内部のアセットなど、一部の情報が失われることに注意してください。コマンド ラインでファイル名を指定しないと、ファイルを選択するよう求められます。EAGLE 7.x 用に保存する場合は、@name 形式が(ロードした図面の名前を変更するために)サポートされないことに注意してください。

### デザイン ブロックの生成および書き出し

#### 一般

オプション DBL を使用すると、現在ロードされている回路図やボード(どのエディタから実行されているかと、回路図とボードが一貫した状態にあるかどうかに応じて異なる)からデザイン ブロックが生成されて、指定した名前で保存されます。名前を指定しないと、デザイン ブロック生成用のダイアログがポップアップされます。また、HTML の説明と属性を入力することもできます。左上に、説明のプレビューがあります。説明は、下のテキスト フィールドに書き出すことができます。左下で属性を管理することができます。また、自動的に生成される属性もあり、これらは編集することができません。右側のプレビューには、デザイン ブロックに含まれる図面が表示されます。下部では、ファイル名を入力したり、ファイルの保存場所を選択することができます。

#### 一貫した選択

プルダウン メニュー エントリの[ファイル]/[選択範囲をデザイン ブロックとして保存]を使用すると、現在の回路図またはボード、あるいはその両方のパーツを選択して、デザイン ブロックとして保存することができます。このメニュー エントリをクリックした後、最初のエディタで選択を行うことができます。選択は加算モードで機能し、数回調整することができます。[Ctrl]を押しながらクリックすることによる選択解除もサポートされています。最初のエディタでの選択を終了するには、[Ctrl]を押しながら右クリックします。このような選択をチェックする基準がいくつかあります(下記を参照)。これらの基準を満たしていない場合は、対応するエラー メッセージが表示され、ユーザは選択を続けて再度[Ctrl]を押しながら右クリックして選択内容を修正することができます。

エディタを 1 つしか開いていない場合は、上記のようなデザイン ブロック ダイアログがポップアップし、現在の選択内容がプレビューに表示されます。選択内容は、このように保存できます。

両方のエディタを開いている場合は、[Ctrl]を押しながら右クリックして、2 番目のエディタで選択を続けることができます。
2 番目のエディタでは、当初、前方注釈/後方注釈に対して最初のエディタの対応するオブジェクトが選択されています(たとえば、最初にパーツ インスタンス R1 を選択していた場合は要素 R1)。2 番目のエディタでの選択では、_一貫性_を妨げない限り、追加でオブジェクトを選択できます。一貫性の妨げになるオブジェクトの選択または選択解除は、自動的にフィルタ アウトされます。たとえば、対応す���ネットが選択されている場合、電気的な意味がない文字や寸法などのオブジェクトを選択したり、信号から配線されたワイヤやポリゴンベタの選択や選択解除ができます。選択を終了するには、[Ctrl]を押しながら右クリックして、デザイン ブロック ダイアログを表示します。これで、結合された選択がプレビューに表示され、後で再利用できるようにデザイン ブロックとして保存できます。
結合された選択が表示されるのは、一貫した回路図とボードのペアがロードされ、両方向に機能する場合のみです。これは、階層デザインではまだサポートされていません。

#### 選択基準

回路図に複数のシートがあり、ボードから選択を開始した場合は、対応物が現在のシート上にあるオブジェクトのみがサポートされます。
回路図から選択を開始した場合は、ユーザがネット セグメント全体を選択しているかどうかをチェックします。特に、ネット ワイヤやラベルが残っていないことを確認します。
パーツごとに、そのすべてのインスタンスを選択する必要があります。
ネット セグメントを選択した状態で、このセグメントに接続されているすべての接続パーツ インスタンスを選択する必要があります。
