## CHANGE

**機能**:

 パラメータを変更します。

**構文**

 CHANGE option ..

**マウス キー**

 *[Ctrl]+右*クリックは、グループのパラメータを変更します。

CHANGE コマンドは、オブジェクトのプロパティを変更またはプリセットするために使用されます。変更を適用するには、CHANGE コマンドのメニューから目的のパラメータを選択するか、キーボードからパラメータを入力して、選択したオブジェクトをクリックします。CHANGE コマンドで調整したパラメータは、後で追加するオブジェクトのプリセット プロパティのままです。

CHANGE コマンドのすべての値は、実際のグリッド単位に従って使用されます。

### グループを変更する

CHANGE コマンドをグループに対して使用する場合、適切なパラメータを指定して CHANGE コマンドを入力する前に、まずグループを [GROUP](ECD-CLI-G.html#group) コマンドで指定します。変更を実行するには、マウスの右ボタンを使用します。

### 文字フォント

**CUSTOM** フォントは、インストールされている **True Type/Clear Type** フォントを使用します。関連付けられた**フォント ファミリ**は、**FONTFAMILY** プロパティを使用して選択できます。

### 文字フォント ファミリ

後続のユーザ テキストの**フォント ファミリ(FONTFAMILY)**を、「new-fontfamily」で指定した値に変更します。**[文字プロパティ]**ダイアログの**[フォント ファミリ]**ドロップダウン メニューに、使用可能なフォント ファミリが表示されます。Microsoft Windows では、**[フォント設定]**に使用可能なフォントが一覧表示されます。MacOS では、**[すべてのフォント]**に使用可能なフォントが一覧表示されます。**フォント ファミリ**は、いずれかのオペレーティング システムでフォントを検索する際に見つかる**[フル ネーム]**フォント プロパティにマップします。

### 例

CHANGE DLINE 0.1mm;

寸法線の幅を 0.1 mm に変更します。延長パラメータは変更されません。

CHANGE DLINE 0.2mm - 0.5mm AUTO;

寸法線の幅を 0.2 mm に、寸法補助線の長さを 0.5 mm に、延長のオフセットを AUTO に変更します。寸法補助線の幅は変更されません。 

CHANGE FONT CUSTOM

は、フォント タイプを CUSTOM に変更します(true type/clear type フォントを使用します)。

CHANGE FONTFAMILY 'Arial'

は現在のフォント ファミリを「Arial」に変更します。注: 引用符が必要です。

## CIRCLE

**機能**

 図面に円を追加します。

**構文**

 CIRCLE .. [center, circumference]
  CIRCLE width ..

**マウス キー**

 *中央*クリックは、レイヤーを選択します。

> **Note:** **関連項目**: [CHANGE](#change)、[LINE](ECD-CLI-L.html#line)

円を作成するには、CIRCLE を使用します。RestrictTop、RestrictBottom、RestrictVias の各レイヤーの円は、制限された領域を定義します。幅を 0 にして定義する必要があります。

width パラメータは、円の円周の幅を定義し、LINE コマンドで使用されるパラメータと同じパラメータです。幅は、次のコマンドを使用して変更できます。

CHANGE WIDTH width;

ここで、*width* は現在の単位の目的の値です。幅が 0 の円が塗り潰されます。

結果として得られる円の半径は 1000 mm に制限されます。

### 例

GRID inch 1;
CIRCLE (0 0) (1 0);

半径 1 インチで、中心が原点にある円を生成します。

## CLASS

**関数**

 ネット クラスを定義および使用します。

**構文**

 CLASS
  CLASS number|name
  CLASS number [ name [ width [ clearance [ drill ] ] ] ] [ number:clearance .. ]

> **Note:** **関連項目**: [NET](ECD-CLI-N.html#net)、[SIGNAL](ECD-CLI-S.html#signal)、[CHANGE](#change)

ネット クラスを定義または使用するには、CLASS を使用します。

パラメータを指定しないと、ネット クラスを定義できるダイアログが表示されます。

number または name のみを指定した場合、指定した number または name を持つネット クラスが選択され、後続の NET コマンドおよび SIGNAL コマンドで使用されます。

number と name の両方を指定した場合、指定した number を持つネット クラスにその後のすべての値が割り当てられ、後続の NET コマンドおよび SIGNAL コマンドで使用されます。name の後のいずれかのパラメータが省略されると、ネット クラスは対応する値を保持します。

number が負の場合、number の絶対値を持つネット クラスがクリアされます。既定のネット クラス 0 をクリアすることはできません。

ネット クラス名は大文字と小文字を区別せずに処理されるため、SUPPLY は Supply や SuPpLy と同じになります。

図面内で複数のネット クラスを使用すると、オートルータがジョブを実行するのに要する時間が長くなります。したがって、使用するネット クラスの数を必要最小限に抑えることには意味があります(ここでは、定義されているネット クラスの数ではなく、ネットまたは信号によって実際に使用されるネット クラスの数のみを考慮しています)。

図面間で切り取りと貼り付けを実行するときに競合が発生しないようにするために、同じネット クラスは、すべての図面で同じ番号で定義することが重要です。

モジュールは、回路図自体と同じ一連のネット クラスを使用しています。

[MODULE](ECD-CLI-M.html#module) のネットに [PORT](ECD-CLI-P.html#port) を介した外部接続がある場合、このポートのネットのネット クラスはモジュール内のネットのネット クラスを上書きします。

オートルータは、全体の幅(幅とクリアランスの合計)の要件で並べ替えて信号を処理し、最大のスペースを必要とする信号が最初に処理されます。バス ルータは、ネット クラス 0 の信号のみを配線します。

既存のネット/信号のネット クラスは、CHANGE コマンドを使用して変更することができます。

### 幅

*width* パラメータは、このネット クラスのすべてのオブジェクトの最小幅を定義します。

### クリアランス

*clearance* パラメータは、このネット クラスのさまざまな信号のオブジェクトと他のネット クラスのオブジェクトの間の最小クリアランスを定義します。

### ドリル

*drill* パラメータは、このネット クラスのすべてのオブジェクトの最小ドリル サイズを定義します(実際に drill パラメータを持っているパッドやビアなどのオブジェクトにのみ適用されます)。

### ネット クラス間のクリアランス

クリアランスが number:clearance という形式で指定されている��合、このネット クラスの信号と指定された番号を持つネット クラスの信号の間の最小クリアランスが定義されます。コマンド

CLASS 3 1:0.6mm 2:0.8mm

ネット クラス 1 と 3 の信号の間に 0.6 mm の最小クリアランスを定義し、ネット クラス 2 と 3 の信号の間に 0.8 mm の最小クリアランスを定義します。number:clearance の番号は、そのネット クラス自体の番号(上の例では「3」)以下である必要があることに注意してください。したがって、

CLASS 3 1:0.6mm 2:0.8mm 3:0.2mm

も有効ですが、

CLASS 3 1:0.6mm 2:0.8mm 3:0.2mm 4:0.5mm

は許可されません。2 つのネット クラス間のクリアランス値「0」は、特定の値がないことを意味します。その場合、関係するネット クラスの値のうち大きい方の値が使用されます。

## CONNECT

**機能**

 ピンにパッドを割り当てます。

**構文**

 CONNECT
  CONNECT [ ALL | ANY ] gate_name.pin_name pad_name..
 CONNECT [ ALL | ANY ] pin_name pad_name..

> **Note:** **関連項目**: [PREFIX](ECD-CLI-P.html#prefix)、[OPEN](ECD-CLI-O.html#open)、[CLOSE](#close)、[SCRIPT](ECD-CLI-S.html#script)

ゲートのピンとライブラリ内の対応するフットプリントのパッドとの関係を定義するには、デバイス編集モードで CONNECT を使用します。まず、PACKAGE コマンドを使用して、どのパッケージを使用するかを定義する必要があります。

パラメータを指定せずに CONNECT コマンドを呼び出すと、接続をインタラクティブに割り当てることができるダイアログが表示されます。

### 1 つのゲートがあるデバイス

デバイスにゲートが 1 つだけ含まれている場合、パラメータ gate_name を省略できます。例:

CONNECT gnd 1 rdy 2 phi1 3 !irq 4 nc1 5...

(注: 「!」は反転データ信号を示すために使用されます)。

### 複数のゲートがあるデバイス

デバイスに複数のゲートがある場合、毎回、gate_name、pin_name、および pad_name を使用してパラメータを入力する必要があります。次に例を示します。

CONNECT A.I1     1  A.I2  2   A.O  3;
CONNECT B.I1     4  B.I2  5   B.O  6;
CONNECT C.I1    13  C.I2  12  C.O 11;
CONNECT D.I1    10  D.I2  9   D.O  8;
CONNECT PWR.gnd  7;
CONNECT PWR.VCC 14;

この場合、なじみ深い 7400 の 4 つの NAND ゲートの接続が割り当てられています。このデバイスには、A、B、C、D、および PWR の 5 つのゲートがあります。ゲート入力の名前は I1 と I2 で、出力の名前は O です。CONNECT コマンドは必要に応じて何度でも繰り返すことができます。このコマンドは、すべてのピン/パッド接続で使用することも、特定のピンでのみ使用することもできます。新しい CONNECT コマンドを使用するたびに、関連するピンの前の条件が上書きされます。

1 つのデバイスに多数の接続がある場合、次の例に示すように、すべての接続を 1 つの呼び出しで指定すると CONNECT コマンドの処理が大幅に速くなることに注意してください。

### 同じピンに接続された複数のパッド

電力増幅器や BGA チップなど一部のパーツは、内部で接続された複数のパッドを持つことができます。これは、熱散逸の改善や高電流への対応のために行われます。CONNECT コマンドは、次のように、関連するすべてのパッド名を空白で区切って(したがって、単一引用符で囲んで)リストすることで、これらのケースを処理することができます。

CONNECT ALL I1 '1 3 5';
CONNECT ANY O1 '2 4 6';

最初の例では、ピン I1 は 3 つのパッド(1、3、5)に接続されています。ピン I1 が回路図内のネットに接続されている場合、3 つのパッドすべてをボード内の対応する信号に明示的に接続する必要があります。
2 番目の例では、キーワード ANY は、パッド 2、4、6 のいずれか(またはすべて)を信号に接続できることを示しています。この内部接続を「ブリッジ」として使用することもできます。これを行うには、たとえば、信号の 1 つのセグメントをパッド 2 に接続し、信号の残りをパッド 6 に接続し、これらの 2 つのパッド間に明示的な外部接続を使用しません。言うまでもなく、ライブラリ パーツをデザインするとき、および CONNECT コマンドで ANY を使用するときに、パーツがそのパッドを通過する横流を処理できることを確認する必要があります。不明な場合は、ALL (既定値であり、省略可能)を使用します。ピン名がキーワード ALL または ANY のいずれかと衝突する場合は、ピン名を単一引用符で囲む必要があります。これらのキーワードのいずれかが CONNECT コマンドに現れると、別のキーワードが現れるまで、後続のすべてのピン/パッド接続に適用されます。

CONNECT 'A' '1' 'B' '2' ANY 'C' '3 4 5 6' 'D' '7 8' ALL 'E' '9 10 11';

[RATSNEST](ECD-CLI-R.html#ratsnest) コマンドと [AUTO](ECD-CLI-A.html#auto) コマンドは、ALL および ANY のケースを適切に処理します。

[接続]ダイアログで[接続]ボタンをクリックすると、選択したピンと選択したパッドの間に接続が作成されます。複数のパッドを選択することができ、その場合、これらのすべてのパッドが選択したピンに接続されます。複数のパッドを選択済みとしてマークするには、通常どおり[Ctrl]キーと[Shift]キーを使用します。
[追加]ボタンは、選択したパッドを現在の接続に追加します。
[切断]ボタンは、選択した接続を削除し、簡単に修正できるように、ピンとパッドを別々のリストに戻して選択されたままにします。切断直後に接続すると、切断前と同じ設定になります(逆も同様)。
接続に複数のパッドが含まれている場合、これらのパッドのいずれかまたはすべてを信号に外部接続する必要があるかどうかを示すアイコンが表示されます。このアイコンをクリックすると、モードが切り替わります。このような接続項目を展開すると、すべてのパッドが個別に一覧表示され、いずれかのパッドを選択した状態で[切断]をクリックすると、そのパッドのみが切断されます。

### ピリオドが含まれているゲート名またはピン名

ゲート名またはピン名にピリオドが含まれている場合、特別な考慮なしで入力します(引用符やエスケープ文字は不要)。

### 例

ed 6502.dev;
prefix 'IC';
package dil40;
connect gnd 1 rdy 2 phi1 3 !irq 4 nc1 5 !nmi 6 \
        sync 7 vcc 8  a0 9 a1 10 a2 11 a3 12 a4 \
        13 a5 14 a6 15 a7 16 a8 17 a9 18 a10 19 \
        a11 20 p$0 21 a12 22 a13 23 a14 24 a15 \
        25 d7 26 d6 27 d5 28 d4 29 d3 30 d2 31 \
        d1 32 d0 33 r/w 34 nc2 35 nc3 36 phi0 37 \
        so 38 phi2 39 !res 40;

コマンドが次の行に継続される場合は、行の末尾に文字「¥」を挿入することをお勧めします。これにより、後続のテキストが EAGLE コマンドと混同されないようにすることができます。パラメータをアポストロフィで囲むことによって、コマンドとパラメータを混同しないようにすることもできます。

## COPY

**関数**

 オブジェクトをコピーします。

**構文**

 COPY ..
 COPY deviceset@[library] [name]
  COPY footprint@[library] [name]
  COPY package3d@[library] [name]
  COPY symbol@[library] [name]  

**マウス キー**

 *[Ctrl]+[C]*は、現在選択されているオブジェクトをコピーします。
 *[Ctrl]+左クリック*は、原点でオブジェクトを選択します。 *[Ctrl]+右クリック*は、グループを選択します。 *中央*クリックは、選択したオブジェクトまたはグループをミラー化します。
 *右クリック*は、選択したオブジェクトまたはグループを回転します。 *[Shift]+右*クリックは、回転方向を反転します。 

> **Note:** **関連項目**: [GROUP](ECD-CLI-G.html#group)、[CUT](#cut)、[PASTE](ECD-CLI-P.html#paste)、[ADD](ECD-CLI-A.html#add)、[INVOKE](ECD-CLI-I.html#invoke)、[POLYGON](ECD-CLI-P.html#polygon)

同じ図面内またはライブラリ間でオブジェクトをコピーするには、COPY を使用します。Electronics ではコピー用の新しい名前が生成されますが、古い値は保持されます。信号(ワイヤ)、バス、およびネットをコピーすると名前が保持されますが、その他の場合はすべて新しい名前が割り当てられます。

### システムのクリップボードにコピーする

従来、EAGLE の COPY コマンドは、オブジェクトをクリックして同じ図面内にインスタンスを配置するか、ライブラリ間でライブラリ オブジェクトをコピーする場合にのみ使用されていました。ただし、バージョン 6 より前のバージョンでは、現在のグループ選択をシステムのクリップボードに送信していませんでした。EAGLE では、このために [CUT](#cut) コマンドが使用されていましたが、バージョン 6 以降では、COPY コマンドは選択したオブジェクトのグループもクリップボードにコピーします。ただし、前バージョンの機能はすべて保持されています。この動作が好ましくない場合は、次の [SET](ECD-CLI-S.html#set) コマンドを使用して、

SET Cmd.Copy.ClassicEagleMode 1

COPY コマンド(および [CUT](#x53) コマンド)の元の動作に戻すことができます。

### ワイヤをコピーする

信号に属するワイヤまたはポリゴンベタをコピーした場合、そのコピーは同じ信号に属します。このため、COPY コマンドを使用した後に 2 本のワイヤが重なっている場合、DRC はエラーを登録しないことに注意してください。
コピーしたワイヤの端点の 1 つがソース セグメント上にある場合、ワイヤはこのセグメントに接続されます。そうでない場合は、このワイヤ用に追加のセグメントが作成されます(「コピーと接続」も参照)。

### パーツをコピーする

回路図内のパーツをコピーすると、マルチゲート パーツの 1 つのゲートのみを選択した場合でも、パーツ全体の新しいインスタンスが必ず追加されます。選択したゲートに加えて、そのデバイスの他のゲートで、Add-Level MUST または ALWAYS を持つものは、自動的に起動されます。マルチゲート パーツの別のゲートを使用するだけの場合、代わりに [INVOKE](ECD-CLI-I.html#invoke) コマンドを使用する必要があります。

### ライブラリ オブジェクトをコピーする

COPY deviceset@[library], COPY footprint@[library], COPY package3d@[library]、または COPY symbol@[library] と記述することにより、デバイス セット、フットプリント、3D パッケージ、または記号を、指定したライブラリから現在ロードされているライブラリにコピーすることができます。library 属性は、プレーンなライブラリ名、ファイル パス([ADD](ECD-CLI-A.html#add) コマンドを参照)、またはダウンロードした管理ライブラリの URN のいずれかで、省略することもできます。ライブラリ名の場合は、現在設定されているライブラリ パスとダウンロードした管理ライブラリの中から検索されます。URN の場合、ダウンロードした管理ライブラリの中から検索されます。library を省略すると、現在ロードされているライブラリがソースとして取得されます。あいまいな場合、オブジェクト名に接尾辞 .dev、.fpt、.p3d、または .sym を追加できます(「.fpt」を使用してフットプリントを指定することは、EAGLE 9.1 の新機能であることに注意してください。旧バージョンとの互換性を保つため、「.pac」もサポートされています)。追加の名前を指定すると、コピーしたオブジェクトにその名前が付けられます。これは、ライブラリ オブジェクトのコンテキスト メニュー、またはコントロール パネルのツリー ビューから_ドラッグ アンド ドロップ_を使用して行うこともできます。**コピーしたライブラリ オブジェクトで使用されている既存のライブラリ オブジェクト(デバイス セット、記号、フットプリント、3D パッケージ)は自動的に更新されます。**

### デザイン オブジェクトをコピーする

COPY device [name]、COPY footprint [name]、COPY package [name]、または COPY symbol [name]と記述することにより、デバイス セット、フットプリント、3D パッケージ、またはシンボルを、デザインから現在ロードされているライブラリにコピーできます。name は、パーツ名(回路図の場合)または要素名(ボードの場合)です。COPY symbol [name]は回路図ウィンドウでサポートされ、COPY footprint [name]と COPY package [name]はボード ウィンドウでサポートされ、COPY device [name]は両方のウィンドウでサポートされます。デバイスは、対応する回路図が使用可能な場合に、ボードか��コピーできます。コマンドで name を指定しない場合、デザインを左クリックしてロードされているライブラリにコピーすることで、有効なオブジェクトを選択できます。

### グループをコピーする

マウスの右ボタンでグループを選択してコピーする操作は、暗黙的な [CUT](#x53) 操作の直後に [PASTE](ECD-CLI-P.html#paste) を実行することにより行われます。

### オブジェクトを他のシートにコピーする

オブジェクトを同じ回路図の別のシートにコピーするには、オブジェクトに対して [GROUP](ECD-CLI-G.html#group) を実行し、[COPY](#copy) (または [CUT](#x53))を実行し、ターゲット シートに切り替えてから [PASTE](ECD-CLI-P.html#paste) を実行する必要があります。

### コピーと接続

ワイヤをコピーしても、接続は変更されません。たとえば、ネットをコピーして別のネットに配置した場合、このネットには接続されません。同じネットやバスなどのセグメントでも同様です。接続の場合は、[NAME](ECD-CLI-N.html#name) または [JUNCTION](ECD-CLI-J.html#x73) を使用します。

### [Ctrl]+[C]とフリー アンカーを使用してコピーする

[Ctrl]+[C]を使用してオブジェクトをコピーすると、1 つのオブジェクトの場合は、複数のオブジェクト(グループ)を使用した場合とは異なる動作をします。1 つのオブジェクトをコピーする場合、後でオブジェクトを貼り付けるときの参照点として原点が使用されます。オブジェクトのグループをコピーする場合、「フリー アンカー」と呼ばれるアンカーを定義できます。
つまり、コピーに使用した選択グループ内のこの点が EAGLE によって記憶され、貼り付け時にその点が使用されます。典型的な例は、大きいオブジェクト グループで、これらのオブジェクト内の特定の点(既定の中心位置ではない)を基準にして配置する場合です。
この場合、選択した場所にマウス カーソルを合わせてから[Ctrl]+[C]を使用すると、この点が貼り付けに使用されます。つまり、オブジェクトのグループの場合、[Ctrl]+[C]はオブジェクトをコピーしたときのカーソルの位置を記憶し、貼り付けに使用します。

## CUT

**関数**

 グループをクリップボードにコピーします。

**構文**

 CUT
  CUT;

> **Note:** **関連項目**: [PASTE](ECD-CLI-P.html#paste)、[COPY](#copy)、[GROUP](ECD-CLI-G.html#group)

図面の一部(またはボード全体)を他の図面にコピーするには、CUT と PASTE を使用します。

まず、グループを定義します(GROUP コマンド)。次に、CUT コマンドと、その後に続く参照点(マウス クリックまたは座標(x y))を使用して、選択したオブジェクトをバッファに入れます。CUT; は、選択したオブジェクトの中心に参照点を自動的に配置します(グリッドにスナップ)。これで、他の図面(EDIT)に変更して、PASTE コマンドを実行してバッファの内容を新しい図面にコピーできるようになりました。

### 参照点

CUT コマンドを選択した後でマウスをクリックすると、マウス カーソルの位置でグループの参照点が定義されます。つまり、PASTE コマンドを使用するときに、マウス カーソルは正確にそのグループの位置にあります。

### 注

他の(Windows の)プログラムとは異なり、Electronics の CUT コマンドでは、マークされたグループは図面から物理的に削除されるわけではありません。このコマンドはグループをクリップボードにコピーするだけです。EAGLE バージョン 6 以降では、CUT コマンドはメイン ドロップダウン メニューまたはツールバーに表示されなくなりました(コマンド ラインおよびスクリプト内で引き続き使用できます)。Windows ユーザは、[COPY](#copy) コマンドを使用して、選択したオブジェクト グループをシステムのクリップボードにコピーします。ただし、これでは選択したグループの明示的な参照点を定義することはできません。参照点は、常にグループの境界長方形の中心で選択されます。参照点の使用は、CUT コマンドでのみ可能です。[SET](ECD-CLI-S.html#set) コマンドを使用することが代替手段になります。

SET Cmd.Copy.ClassicEagleMode 1

CUT コマンド(および [COPY](#copy) コマンド)の元の動作に戻すことができます。
