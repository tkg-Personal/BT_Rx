## TEXT

**関数**

 図面にテキストを追加します。

**構文**

 TEXT any_text [orientation] ..
 TEXT 'any_text' [orientation] ..

**マウス キー**

 *中央*クリックは、レイヤを選択します。
 *右*クリックはテキストを回転します。
 *[Shift]+右*クリックは、回転方向を反転します。

**関連項目**: [CHANGE](ECD-CLI-C.html#change)、[MOVE](ECD-CLI-M.html#move)、[MIRROR](ECD-CLI-M.html#mirror)、[PIN](ECD-CLI-P.html#pin)、[ROTATE](ECD-CLI-R.html#rotate)、[ATTRIBUTE](ECD-CLI-A.html#attribute)

TEXT コマンドは、ライブラリ要素または図面にテキストを追加するために使用されます。複数のテキストを入力する場合、マウスでテキストを配置した後も TEXT コマンドがアクティブのままなので、コマンドを毎回呼び出す必要がありません。

### 方向

テキストの方向は、[ADD](ECD-CLI-A.html#add) コマンドで指定するような通常の定義(R0 や R90 など)を使用して、TEXT コマンド(orientation)で定義できます。マウスの右ボタンでテキストの回転を変更し、マウスの中央ボタンで現在のレイヤを変更します。テキストが常に表示されているため、回転させても、正面または右からテキストを読み取ることができます。そのため、回転を 2 度行うたびに、何も変わっていないように見えますが、原点は左下から右上隅に移動しています。テキストが選択できないように見える場合は、このことを思い出してください。

縦書きのテキストの読み取り方向は、ユーザ インタフェース ダイアログで「上」から「下」に変更できます。

テキストを「上下反転」に印刷する場合は、そのテキストに「スピン」フラグを設定できます。

### マルチ テキスト

テキストには改行を含めることができます。テキスト ダイアログでは、[Shift]+[Enter]を押して改行を直接入力することができます。コマンドラインでは改行文字「\n」で改行する必要があります。

### 特殊文字

テキストに複数の連続する空白またはセミコロンが含まれている場合は、文字列全体を単一引用符で囲む必要があります。テキストに単一引用符が含まれている場合は、各引用符自体を単一引用符で囲む必要があります。テキストにアポストロフィが必要な場合は、各アポストロフィを単一引用符で囲む必要があります。
テキストに円記号を含める場合は、各円記号を別の円記号でエスケープする必要があります。(たとえば、TEXT 'abc\\def' とすると、abc\def が生成されます)。

### キーワード

TEXT コマンドがアクティブで、入力するテキストにコマンドと間違えられる文字列が含まれている場合は(「REDO」の「red」など)、この文字列を単一引用符で囲む必要があります。

### 文字の高さ

文字の高さと厚さは、CHANGE コマンドで変更することができます。

CHANGE SIZE text_size ..
CHANGE RATIO ratio ..
CHANGE LINEDISTANCE value ..

文字の高さの最大値: 2 インチ
文字の厚さの最大値: 200 mm (7.7840 インチ)
比率: 0 ～ 31 (文字の高さの %)
行間隔: 0 ～ 250 (文字の高さの %)。

### 文字フォント

テキストには、次の 3 種類のフォントを使用することができます。

| 

Vector | 

 | 

プログラムの内部ベクトル フォント | 

 |

| 

比例(Proportional) | 

 | 

プロポーショナル ピクセル フォント(通常は「Helvetica」) | 

 |

| 

固定 | 

 | 

等幅ピクセル フォント(通常は「Courier」) | 

 |

文字のフォントは、CHANGE コマンドで変更できます。

CHANGE FONT VECTOR|PROPORTIONAL|FIXED ..

プログラムは、できる限りベクトル以外のフォントを使用してテキストを出力しようとします。ただし、実際のフォントはシステムのグラフィックス インタフェースによって描画されるため、プロポーショナル フォントと固定フォントは異なるサイズや長さで出力される場合があります。ユーザ インタフェース ダイアログで[常にベクトルフォント]オプションを設定すると、個々の文字や図面ごとに設定した設定に関係なく、すべての文字は常に組み込みのベクトル フォントを使用して表示および印刷されます。このオプションは、システムが他のフォントを正しく表示しない場合に便利です。
[SET](ECD-CLI-S.html#set) インタフェースを使用できます。この設定を変更するには、VectorFont 0|1 コマンドを使用します。

新しいボードまたは回路図を作成する場合、このオプションの現在の設定は、図面ファイルの[この図面で保持]オプションに保存されます。これにより、このオプションの設定が異なるユーザに図面が転送された場合でも、図面が正しい設定で印刷されるようになります。
[SET](ECD-CLI-S.html#set) VECTOR_FONT OFF|ON コマンドを使用すると、既存のボードまたは回路図図面の設定を変更できます。

CAM プロセッサで出力ファイルを作成する場合、テキストは常にベクトル フォントで描画されます。その他のフォントはサポートしません。

フォントがベクトル以外のテキストが信号銅箔ベタから差し引かれる場合、周囲の長方形のみが差し引かれます。前述したサイズや長さの問題が発生する可能性があるため、実際に印刷されるフォントがこの長方形よりも大きくなる可能性があります。したがって、信号ポリゴンベタからテキストを差し引く必要がある場合は、ベクトル フォントを使用することをお勧めします。

*Ratio* パラメータは、フォントがベクトル以外のテキストに使用しても意味がありません。

### 文字の位置合わせ

テキストの位置合わせでは、原点をテキスト内に配置する場所を定義します。位置合わせには 9 種類の方法があり、左、下、中心、右、上の各キーワードの組み合わせで構成されています。これらのキーワードは任意の順序で指定できますが、左/右および上/下のうち最後のキーワードのみがそれぞれの方向に考慮されます。中心キーワードは、他のキーワードが指定されていない方向にのみ適用されます。既定は、左と下です。

| 

CHANGE ALIGN TOP CENTER; TEXT 'ABC'  | 

 | 

原点を上/中心にして ABC というテキストを描画します。 | 

 |

### キャラクタ セット(Character Sets)

ASCII コードが 128 以下の文字のみが正しく印刷されるようになります。これより上の文字は、システムに依存する可能性があり、フォントによって異なる結果になる可能性があります。

### 属性

記号またはフットプリント図面に実際のパーツまたは要素の属性を表示する場合は、属性の名前が「>」文字でマークされたテキストを使用することができます。既定では、指定した属性の実際の値のみが表示されます。属性名の後に「=」、「~」、「!」のいずれかの特殊文字を続けると、実際の表示は次のようになります。

| 

>ABC | 

 | 

123 | 

 |

| 

>ABC= | 

 | 

ABC = 123 | 

 |

| 

>ABC~ | 

 | 

[ABC] | 

 |

| 

>ABC! | 

 | 

_nothing_ | 

 |

属性名ごとに、指定した記号またはフットプリント内にこのようなテキストが 1 つのみ存在する必要があります。記号またはフットプリントに同じ属性名を参照するテキストが複数ある場合に、この記号またはパッケージを使用するパーツが分解されていると、それらのテキストの 1 つのみが表示されます。

### テキストの上線

テキストに_上線_を付けることができます。これは、反転信号(「active low」、[NET](ECD-CLI-N.html#net)、[BUS](ECD-CLI-B.html#bus)、[PIN](ECD-CLI-P.html#pin) も参照)の名前などに便利です。そのためには、次のように、テキストの前に感嘆符(「!」)を付ける必要があります。

  !RESET

これは、次のような結果になります。

  _____
  RESET

これは信号名に限らず、任意のテキストで使用することができます。次のように、テキストの一部にのみ上線を付けることもできます。

  !RST!/NMI
  R/!W

これは、次のような結果になります。

  ___
  RST/NMI
    _
  R/W

2 番目の感嘆符は上線の最後を示します。1 つのテキスト内に任意の数の上線を付けることができます。上線を生成しない感嘆符をテキストに含める場合は、円記号を使用してエスケープする必要があります。感嘆符をエスケープする必要性を最小限に抑えるため、感嘆符がテキストの最後の文字である場合や、感嘆符の直後に空白、二重引用符、単一引用符、または右括弧が続く場合には、感嘆符は上線を開始しません。上線を開始した感嘆符の後にエスケープしない感嘆符またはカンマが続く場合、上線は終了します(バスでは、上線を終了する記号としてカンマが必要です)。
