## VALUE

**関数**

 値を表示および変更します。

**構文**

 VALUE ..
 VALUE value ..
 VALUE name value ..
 VALUE ON;
  VALUE OFF;

**関連項目**: [NAME](ECD-CLI-N.html#name)、[REPOSITION](ECD-CLI-R.html#reposition)、[VARIANT](#variant)

### ボードと回路図の場合

要素に値を割り当てることができます。たとえば、抵抗器の場合は「1k」、コンデンサの場合は「10uF」とします。これを行うには、VALUE コマンドを使用します。このコマンドを実行すると要素が選択され、値を入力または変更できるポップアップメニューが開きます。要素を選択する前に値を入力すると、その後選択したすべての要素にその値が適用されます。これは、たとえば多くの抵抗器に同じ値を設定する場合に非常に便利です。

パラメータの名前と値を指定すると、指定した値が要素名に適用されます。

VALUE コマンドは、既定のアセンブリ バリアントでのみ使用することができます。別のアセンブリ バリアントの値を変更する場合は、[VARIANT](#variant) コマンドを使用する必要があります。

### 例

VALUE R1 10k R2 100k

この場合は、複数の要素に値が割り当てられます。この操作をスクリプト ファイルで行うには次のようにします。

VALUE R1   10k \
      R2  100k \
      R3  5.6k \
      C1  10uF \
      C2  22nF \
      ...

「\」は、後続の行が EAGLE のキーワードと間違われるのを防ぎます。

### デバイス モードの場合

VALUE コマンドをデバイス編集モードで使用する場合、パラメータ ON および OFF を使用することができます。ON にすると、回路図で実際の値を変更することができます。

OFF にすると、回路図に実際のデバイス名(74LS00N など)が自動的に入力されます。この値は、確認して初めて変更されます。

## VARIANT

**関数**

 アセンブリ バリアントを管理します。

**構文**

 VARIANT
  VARIANT name part_name [NO]POPULATE [ value [ technology ] ];
  VARIANT [ + | - ] name;

**関連項目**: [VALUE](#value)、[TECHNOLOGY](ECD-CLI-T.html#technology)

既定では、回路図のすべてのパーツがボードに入力されます(パッケージがある場合)。ただし、1 つのデザインにさまざまなバリアントを設定するためには、一部のパーツを入力しないようにしたり、パーツの値やテクノロジを既定とは異なるものにすることが必要になる場合があります。VARIANT コマンドを使用すると、特定のアセンブリ バリアントに実際にどのパーツを入力するかを定義し、そのパーツに特定の値とテクノロジを指定することができます。

*name* はバリアントの名前です。この値は大文字と小文字が区別されず、空白、「+」または「-」を含める場合には単一引用符で囲む必要があります。

*part_name* の後にキーワード POPULATE または NOPOPULATE を続けた場合は、そのパーツのバリアントが指定の *name* で作成され、「入力される」パーツまたは「入力されない」パーツとしてマークされます。

現在のアセンブリ バリアントに入力されていないパーツは、回路図内の境界領域全体に X が描画されます。このような場合ボードでは、実際にパーツをボードに配置することに関連するもの(配置プラン、名前、値など)はすべて描画されません。

オプションの *value* と *technology* (回路図でのみ使用可能)を使用して、バリアントをさらに調整できます。value を指定できるのは、パーツのデバイス セットで「user value」パラメータが有効になっている場合のみです。value に別の値を使用せずに technology のみを指定する場合は、value に空の文字列(''')を入力することができます。

新しいバリアントを作成するには、名前の前に「+」を付けます。その名前のバリアントが既に存在する場合は、何も起こりません。

バリアント名の前に「-」を付けた場合は、指定したバリアントが削除されます。name を「*」にすると、すべてのバリアントが削除されます。このコマンドをスクリプトで使用していない場合は、確認プロンプトが表示され、このアクションを実際に実行するかどうかを尋ねられます。

バリアント名のみを指定すると、プロジェクト全体がそのバリアントに切り替わります。つまり、すべての「populate」フラグ、値、およびテクノロジが、パーツごとにそのバリアントに指定したとおりに表示されます。ここで空の文字列('')を使用すると、既定のアセンブリ バリアントに切り替わります。これは、バリアントがまったくない場合と同じです。図面をロードしたとき、図面は常にアセンブリ バリアントが選択されていない既定の状態になります。

パラメータを指定せずに使用した場合は、すべてのアセンブリ バリアントを管理できるダイアログが開きます。

現在のアセンブリ バリアントの名前は、テキスト変数 >ASSEMBLY_VARIANT を使用して表示することができます。

ADD、CHANGE PACKAGE | TECHNOLOGY、REPLACE、UPDATE、VALUE の各コマンドは、アクティブなアセンブリ バリアントがない場合にのみ使用することができます。

COPY コマンドは、アセンブリ バリアントをコピーしません。

ボードでは、VARIANT コマンドがサポートされていません。前方注釈/後方注釈がアクティブな場合は、すべてが回路図からコントロールされます(ボード エディタのバリアント コンボ ボックスは、情報を提供する目的でのみ使用されます)。ただし、スタンドアロン ボードの場合は、[CHANGE](ECD-CLI-C.html#change) コマンドまたはプロパティ ダイアログで要素の入力オプションを設定できます。

### モジュールのアセンブリ バリアント

モジュールのアセンブリ バリアントを作成することもできます。モジュールを参照する各モジュール インスタンスで、そのモジュールの特定のバリアントを選択できます。回路図のアセンブリ バリアントとモジュールのバリアントは、完全に独立しています。VARIANT コマンドは、モジュールのアセンブリ バリアントを切り替えるためのものではありません。

このため、より明確にするために、***モジュール アセンブリ バリアント***と***メイン アセンブリ バリアント***を区別します。VARIANT コマンドは、モジュール内ではモジュール アセンブリ バリアントを参照し、回路図のメイン シートではメイン アセンブリ バリアントを参照します。

## VIA

**関数**

 ボードにビアを追加します。

**構文**

 VIA ['signal_name'] [diameter] [shape] [layers] [flags] ..

**関連項目**: [SMD](ECD-CLI-S.html#smd)、[CHANGE](ECD-CLI-C.html#change)、[DISPLAY](ECD-CLI-D.html#display)、[SET](ECD-CLI-S.html#set)、[PAD](ECD-CLI-P.html#pad)、デザイン ルール

VIA コマンドがアクティブな場合、ビア記号がカーソルにアタッチされます。マウスの左ボタンを押すと、現在の位置にビアが配置されます。ビアは、既存の信号ワイヤに配置されている場合には信号に追加されます。別の信号を接続しようとすると、本当に接続してよいかどうかを尋ねられます。

### 信号名

signal_name パラメータは主に、生成されたデータを読み込むスクリプト ファイルで使用することを目的としています。signal_name を指定すると、後続のすべてのビアがその信号に追加され、自動チェックは実行されません。
**この機能を使用する場合は細心の注意を払ってください。別の信号に属するワイヤを接続するようにビアが配置されている場合、短絡が発生する可能性があるためです。signal_name パラメータを指定して VIA コマンドを使用した後で、デザイン ルール チェックを実行することをお勧めします。**

### ビアの直径

数値を入力すると、ビアの直径が(実際の単位で)変更され、その値は引き続き他のビアに使用されます。ビアの直径は、最大 200 mm (7.7840 インチ)まで指定することができます。ビアのドリル直径は、パッドに設定された直径と同じです。変更するには、次のコマンドを使用します。

CHANGE DRILL diameter 

### 形状

ビアの形状は、次のいずれかにすることができます。正方形
円形
八角形

これらの形状は、外側画層(Top と Bottom)にのみ適用されます。内側画層では、形状は常に「円形」です。

ビアは、Drills 画層にドリル記号を生成し、SolderMaskTop/SolderMaskBottom 画層にはんだストップ マスクを生成します。

直径と同様に、VIA コマンドがアクティブな間にビアの形状を入力したり、CHANGE コマンドを使用してビアの形状を変更することができます。その形状は、次のビアやパッドにも有効です。

ビアの実際の形状と直径は、ビアが使用されているボードのデザイン ルールによって決まることに注意してください。

### レイヤ

layers パラメータでは、このビアがカバーする画層を定義します。構文は「from-to」で、「from」と「to」はカバーされる画層番号です。たとえば、2-7 とすると、画層 2 から画層 7 までのビアが作成されます(7-2 も同じ意味になります)。そのような正確なビアがデザイン ルールのレイヤ設定で使用できない場合は、次に長いビアが使用されます(そのようなビアを設定できない場合には、エラー メッセージが表示されます)。[SET](ECD-CLI-S.html#set) **Option.ViaLength 1** を使用すると、ビアの長さがテキストで表示されます。

### フラグ

次の_フラグ_を使用すると、ビアの外観をコントロールできます。

| 

STOP | 

 | 

常にはんだストップ マスクを生成します。 | 

 |

既定では、ドリル直径がデザイン ルールのパラメータ「Masks/Limit」の値以下のビアには、はんだストップ マスクがありません。上記の STOP フラグを使用すると、ビアにはんだストップ マスクを強制的に設定することができます。

## VPPROBE

**関数**

 電圧位相プローブをバスおよびネットにアタッチします。

**構文**

 VPPROBE

**関連項目**: [NET](ECD-CLI-N.html#net)、[BUS](ECD-CLI-B.html#bus)

電圧位相プローブを使用すると、ユーザは AC シミュレーション結果に節点電圧の位相を表示することができます。コマンドは、回路図内のネットのコンテキスト メニュー項目、コマンド ライン、およびツールバーのコマンド ボタンを介して使用できます。
