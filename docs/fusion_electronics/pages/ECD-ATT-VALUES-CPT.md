# 属性と属性値

**属性**は、Fusion 電子デザインのライブラリ、コンポーネント、またはコンポーネント バリアントを記述するプロパティです。**属性値**は、製造元名、パーツ番号、説明などのプロパティのデータを格納します。

属性とその値によって、ライブラリ全体でのコンポーネント データの保存、継承、オーバーライドの方法が定義されます。

## 属性とは

属性は、**MANUFACTURER**、**MPN**、**DESCRIPTION** などのプロパティのタイプを定義します。属性を作成するときは、属性スコープを選択して、属性の適用場所を定義します。

いったん作成されると、属性はそのスコープに存在し、その中のすべてのアイテムに使用できます。

## 属性スコープ

属性スコープは、属性値の適用場所を定義します。

- **ライブラリ**: ライブラリ内のすべてのコンポーネントとバリアントに値を適用します。

- **コンポーネント**: 現在のコンポーネントのすべてのバリアントに値を適用します。

- **バリアント**: 値を現在のバリアントに適用し、すべての属性セットに既定値を指定します。

- **属性セット**: 現在のバリアントの選択した属性セットにのみ値を適用します。

## 属性値

属性値は、属性に割り当てられたデータです。属性値は[属性]ダイアログ ボックスにセルとして表示され、直接編集できます。

属性値は、スコープに基づいて相互に継承およびオーバーライドします。これらの値をさまざまなスコープ レベルで編集して、ライブラリ、コンポーネント、およびバリアント全体にどのように適用するかをコントロールできます。

下位スコープは上位スコープをオーバーライドします。これにより、既定値を広範に定義し、必要に応じてオーバーライドできます。

**例:**

ライブラリ スコープで MANUFACTURER の値を "My Company" に設定して、グローバル既定を作成します。特定のコンポーネントについて、コンポーネント スコープの値を "Other Company" でオーバーライドします。これらの特定のコンポーネントは "Other Company" に解決され、ライブラリの残りの部分は引き続き "My Company" を使用します。

## 属性のタイプ

Fusion 電子デザインでは、次の 2 つの属性タイプがサポートされています。

- **変数**: 属性値を回路図でオーバーライドできます。

- **定数**: 属性値を回路図でオーバーライドできません。
