# オートルータのコストと効果

これらは、オートルータのパラメータと、それに関連するコストと効果です。パラメータの名前は、オートルータ コントロール ファイル(.ctl)で使用される名前と同じです。

## 画層コスト

| 
 | 
 | 
 |

| 
cfBase.xx: 0..20 | 
対応する画層上の 1 ステップの基準コストです。推奨: 外側画層(Top、Bottom)は常に 0、内側画層は 0 よりも大きくします。 | 
 |

## コスト

| 
 | 
 | 
 |

| 
cfVia: 0..99 | 
ビアの使用をコントロールします。値を小さくすると、多くのビアが生成されますが、優先方向に従うこともできます。値を高くすると、ビアを回避しようとして、優先方向に違反する可能性があります。推奨: 配線パスの値を低くし、最適化の値を高くします。 | 
 |

| 
cfNonPref: 0..10 | 
優先方向の準拠をコントロールします。値を低くすると、トラックを優先方向とは逆方向に配線することができ、値を高くすると、優先方向に強制的に配線することができます。
 cfNonPref を 99 に設定すると、トラック セクションを優先方向にのみ配置することができます。この値は、この動作が本当に必要であると確信している場合にのみ選択してください。 | 
 |

| 
cfChangeDir: 0..25 | 
方向の変更頻度をコントロールします。値を小さくすると、トラック内で多くの曲げが許可されます。値を大きくすると、ほぼ直線のトラックが生成されます。 | 
 |

| 
cfOrthStep、cfDiagStep | 
直角三角形の斜辺が他の 2 辺の合計よりも短いというルールを実装します。既定値は 2 と 3 です。つまり、他の 2 辺を使用するルートのコストは 2+2 であり、それに対して斜辺は 3 になるということです。これらのパラメータを変更する場合は、細心の注意を払ってください。 | 
 |

| 
cfExtdStep: 0..30 | 
45 度の角度で優先方向に向き、かつ基板を 2 つのセクションに分割することになる、トラック セクションの回避をコントロールします。値を低くすると、このようなセクションが許可され、値を高くすると、その回避が試みられます。
mnExtdStep パラメータと組み合わせると、これらのトラックの長さをコントロールすることができます。mnExtdStep を 0 にすると、45 度で優先方向に向かう各グリッド ステップでは、cfExtdStep パラメータに定義されているコストが発生します。たとえば、mnExtdStep で 5 を選択すると、追加のコストなしでトラックが 45 度で 5 ステップ進みます。それ以上のステップを進むたびに、cfExtdStep に定義されているコストが発生します。このようにして、90 度の曲げによって 45 度のコーナーが作られます。cfExtdStep を 99、mnExtdStep を 0 のように設定すると、角度が 45 度のトラックが回避されます。このパラメータは、優先方向がある画層にのみ意味を持ちます。
推奨: 配線パスには小さい値を使用し、最適化には大きい値を使用します。 | 
 |

| 
cfBonusStep, cfMalusStep: 1..3 | 
レイアウト内の優先(ボーナス)領域と不良(マリュス)領域との区別を強化します。値を高くすると、ルータは優良領域と不良領域を強く区別します。低い値を使用すると、この係数の影響が低減されます。cfPadImpact と cfSmdImpact も参照してください。 | 
 |

| 
cfPadImpact, cfSmdImpact: 0..10 | 
パッドおよび SMD の周囲には優良と不良のセクションまたは領域が生成され、オートルータはそこにトラックを配置することを優先します(あるいは優先しません)。優良領域は優先方向(定義されている場合)にあり、不良領域は優先方向に対して垂直になります。つまり、優先方向に走行するトラックは、パッドや SMD から離れて配線されます。値を高くすると、トラックはできる限り優先方向に走行しますが、値を低くすると、優先方向からすぐに離れてしまう可能性があります。高密度の SMD 基板の場合、cfSmdImpact の値を若干高くすると効果的な場合があります。 | 
 |

| 
cfBusImpact: 0..10 | 
バス接続の理想線が追従するかどうかをコントロールします(cfPadImpact も参照してください)。値を高くすると、始点と終点間の直結線が追従するようになります。バス配線に対してのみ重要です。 | 
 |

| 
cfHugging: 0..5 | 
平行トラックの沿進をコントロールします。値を高くすると、強力な沿進が可能になり(トラック同士がかなり近接します)、値を低くすると、幅広い分散が可能になります。推奨: 配線の値を高くし、最適化の値を低くします。 | 
 |

| 
cfAvoid 0..10 | 
リップアップの際、トラックが削除された領域は回避されます。値が大きいほど、強い回避を意味します最適化パスには関係ありません。 | 
 |

| 
cfPolygon 0..30 | 
銅箔ベタが RATSNEST コマンドで処理されていて、そのためにオートルータを起動する前に塗り潰し領域として表示される場合は、銅箔ベタ内のすべてのステップがこの値に関連付けられます。
 値を小さくすると、オートルータが銅箔ベタ領域内でトレースの配線を簡単に行えるようになります。ただし、銅箔ベタが複数の要素に分割される可能性が高くなります。値を大きくすると、オートルータによってポリゴンベタ内に作成される接続が少なくなります。ポリゴンベタがアウトライン モード中で、オートルータを起動する前に RATSNEST によって処理されない場合、そのポリゴンベタはまったく考慮されません。cfPolygon はこのようなポリゴンベタに対して機能しません。 | 
 |

## 最大値

| 
 | 
 | 
 |

| 
mnVia 0..30 | 
連結トラックの作成に使用できるビアの最大数をコントロールします。 | 
 |

| 
mnSegments 0..9999 | 
1 つの連結トラック内のワイヤ要素の最大数を決定します。 | 
 |

| 
mnExtdSteps 0..9999 | 
cfExtdStep の値を発生させることなく、優先方向に 45 度で許容されるステップの数を指定します。cfExtdStep も参照してください。 | 
 |
